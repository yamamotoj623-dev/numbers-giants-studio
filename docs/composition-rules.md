★ Gem の Knowledge File として参照される。台本構成設計時の具体的なルール・チェックリストを集約。
★ 縦長 9:16(ショート)と横長 16:9 の両方をカバー。§ 1-3 と §5 以降は ★縦長 9:16 用★、§ 4 が ★横長 16:9 用★。

## 1. id:1 戦略の 4 タイプ運用(★縦長 9:16 のみ★)

動画題材に応じて 1 つ選択。**戦略 D が本命候補**(YouTube ショート「動画感」回避でスワイプ抑制)。

| 戦略 | 用途 | 視覚 | textSize | scenePreset | se |
|---|---|---|---|---|---|
| **A: 派手型** | バズ狙い・新規流入 | 派手画像 + グラデ大タイトル | xl | cinematic_zoom or neon_burst | hook_impact |
| **B: 数字主役型** | 異常値の数字勝負 | 黒背景 + 巨大数字 + 細い文脈 | xl(数字)+ l(補足) | mono_drama | stat_reveal |
| **C: 静物写真型** | 訃報・怪我・落ち着いた内容 | 静物写真 + 控えめ文字 | l | blackboard | soft_pop |
| **D: ぬるっと型** | 通議論・コアファン深掘り | id:2 と同じ普通シーン、派手要素なし | m (or s) | default | なし or soft_pop |

### ★戦略 D(ぬるっと型)の本質特徴★

★「動画が始まった」と気づかれないように始める★。視聴者がスワイプ判断する前に「会話の途中?」と思って 1-2 秒留まる。これがコアファン狙いの最強パターン。

#### 話法ルール

| 項目 | 戦略 A/B/C | 戦略 D |
|---|---|---|
| **断定** | 「○○の真実!」 | ★断定しない★ 「○○、これ変じゃないですか?」 |
| **語尾** | 「!」「!?」で煽る | 「...」「?」「ですね」で静か |
| **強調記号** | 【】「」『』を必ず複数使う | ★強調記号は使わない or 最小限★ |
| **主語** | 選手名から始まる | キャラの呼びかけ or 独り言から始まる |
| **発話順** | A が宣言 or B が代弁 | ★B が「あれ?」と振る → A が「ええ、私も…」と相槌★ |

#### speech のトーン

- 「!」「!?」は基本使わない(静かな違和感が壊れる)
- 「...」「ですね」「ますね」で間を取る
- 「えっ?」のような大きな反応も使わない、代わりに「あ…」「これ…」のような小さな違和感
- 数原(A)は「ええ、気になっていまして」「私も思っていました」のような ★相槌が先★

#### 視覚要素

- emoji は ★控えめ★(🤔 / 🧐 / 😯 / 😌 中心、🤯 / 😆 / 🤩 などの派手系は使わない)
- scenePreset は `default` 固定(neon_burst / cinematic_zoom などの派手系 NG)
- zoomBoost なし
- se は無し or `soft_pop` のみ(hook_impact / shock_hit / stat_reveal は使わない)

#### 適合テーマ(★戦略 D が機能する題材★)

戦略 D は ★全テーマで使えるわけではない★。以下のような ★「コアファンが既に気付いていること」★ を ★言語化する★ 場面で機能する:

| テーマパターン | 例 |
|---|---|
| 数字の矛盾 | 「Stuff+ 107.3 なのに防御率 9.00」「打率.228 なのに OPS .800」 |
| 違和感のある活躍/不振 | 「直球被打率.375 なのに無失点」「打順だけが不自然」 |
| ファンのモヤモヤ | 「球は来てるのに勝てない」「采配は変わってないのに結果が違う」 |
| 通好みの細部 | 「フォーク球数が前年比 2 倍」「初球ストライク率の急上昇」 |

★戦略 D に向かないテーマ★:
- 大型 FA / トレード / 朗報(派手系 A/B が合う)
- 訃報・怪我(C が合う)
- ミーハー向け話題(A が合う)

### ★成功パターン例(戦略 D)★

```
id:1, B (もえか): 「数原さん…\nこれ、ちょっと\n変じゃないですか?」
  textSize: m / scenePreset: default / emoji: 🤔 / se: なし
id:2, A (数原):  「ええ、私も\n気になっていまして」
  textSize: m / scenePreset: default / emoji: 👨‍🏫 / se: soft_pop
id:3, A (数原):  「中川皓太投手の\n直球の話なんですが」
```

→ 視聴者は ★「会話の続きを聞いている」感覚★ になり、1-2 秒留まる。そこで id:4-5 で核心数字を見せると ★期待値回収+引き込み完了★。

### ★失敗パターン例(戦略 D 失格)★

❌ 失敗例 1: 断定してしまう
```
id:1, B: 「中川皓太、衝撃の事実!」  ← 派手系(戦略 A)になっている
```

❌ 失敗例 2: 強調記号使用
```
id:1, B: 「数原さん、【あれ】って…『変』ですよね?」  ← 強調記号で派手化、ぬるっと感喪失
```

❌ 失敗例 3: 「!?」連発
```
id:1, B: 「数原さん!?これ変ですよね!?」  ← 動画感が出てしまい戦略 D の特徴消失
```

❌ 失敗例 4: 戦略 D に不向きなテーマ
```
id:1, B: 「岡本、4HR でサヨナラ!?」  ← 派手な朗報は戦略 A 向き、D でやると弱い
```

## 2. id:1 タイトル設計の 4 原則

### [原則 1] 1 選手フォーカス必須
- ✅ 「岡本和真、メジャー速球を粉砕した『14cm』の秘密」(維持率 60.0%)
- ✅ 「平山功太、求めていた姿すぎる…【打球速度171km/h】」(維持率 83.7%)
- ❌ 「松本剛 vs 佐々木俊輔、阿部監督を悩ませる…」(維持率 22.4%)
- ❌ 「○○の穴を埋める△△」(主役分散型 維持率 15.4%)
- ※ 例外: スタメン争い・ポジション争いを正面から扱う回のみ 2 名 OK

### [原則 2] 具体数値を必ず含める
- ✅ 「打球速度 171km/h」「OPS1.522」「WHIP 0.97」「.333」「14cm」
- ❌ 「驚愕のデータ」「異常な数値」「本当の理由」(抽象数値表現)
- ※ 数値が出せない時は物語キーワード(原則 3)で代替

### [原則 3] 物語キーワードで興味曲線を作る
- ✅ 採用語: 別人 / 復活 / 逆襲 / 覚醒 / バケモノ化 / 現在地 / 真実 / 正体 / 完全復活 / 異次元 / 眩しい
- ✅ 例: 「田中マー君、去年とは別人」(+5 sub) / 「泉口、バケモノ化」(維持率 79.7%) / 「中山礼都の逆襲」(77.1%)
- ※「現在地」は id:1 タイトル限定で解禁(台詞内では引き続き使わない)

### [原則 4 ★最優先★] 「数字 + 矛盾」型を最優先で検討
視聴データ Top 維持率の最強パターン。「○○なのに/なぜ」型。
- ✅ 「赤星優志、防御率1.72の衝撃 なのになぜ便利屋?」(ベストプラクティス)
- ✅ 「石川達也、防御率13.50なのに2軍では無双状態!?」(同)
- ✅ 「戸郷翔征、Stuff+ 107.3 なのに防御率 9.00 の謎」
- ❌ 「もう誰も打てません…!」(結論先出し型 — スワイプ誘発)
- ❌ 「大苦戦する理由」(抽象結論型 — スワイプ誘発)

### タイトル長
- **推奨: 25-32 字**(高維持率動画の中央値)
- **35 字以上 NG**(低維持率特性)

### タイトル推奨型サマリ

| 型 | 構造 | 例 |
|---|---|---|
| A | 選手 + 具体数値 + 比喩 | 「平山功太が『浅野・萩尾』に求めていた姿すぎる…【打球速度171km/h】」 |
| B | 選手 + 物語キーワード + 「件」/「理由」 | 「巨人の正遊撃手・泉口友汰のデータが完全に『バケモノ化』している件…」 |
| C | 権威の引用 + 選手 + 疑問形 | 「江川・野村も苦言、岸田の配球問題…なぜ打たれる?」 |
| D | 選手 + 物語的状態 + ギャップ | 「中山礼都の逆襲!1割台から覚醒するための『驚愕のデータ』とは!?」 |
| **E ★最強★** | **数字 + 矛盾型** | 「赤星優志、防御率1.72の衝撃 なのになぜ便利屋?」 |

## 3. 視聴データ実証ベース(60本動画分析)

| 検証項目 | 結果 |
|---|---|
| 具体数値タイトル | 平均維持率 **+7.9pt** |
| 1選手フォーカス | +4.6pt |
| 阿部監督批判型 | **-3.2pt(逆効果)** |
| 他球団選手比較型 | 維持率 17-18% |
| パワーワード(致命的/絶望/衝撃) | +0.8pt(効果ゼロ、飽和) |

Top 4 維持率動画(83.7% / 79.7% / 77.1% / 70.4%)の共通点:
**1人の選手 + 具体数値 or 物語キーワード + 短いタイトル(43-64字)**

## 4. ストーリー構成
- **基準**: 合計再生時間が 60 秒以内(ショート枠)に収まる範囲
- **目安**: id 20-30 個(60 秒未満ショート) / 25-35 個(60-90 秒) / 35-50 個(90 秒〜2 分)
- ★必須数の指定はしない★ — 内容に応じて柔軟に。無理に水増しした 30 個より、密度の高い 22 個の方が良い

### 標準構成(ミステリー型展開、id 数に応じて伸縮)
```
id:1         主語+ギャップ提示(謎残す、結論言わない) ★4 原則厳守★
id:2-3       状況設定 + A↔B 呼び合い両方向
id:4-7       ★核心データ最速提示(期待値の回収)
id:8-(N-8)   多角的に深掘り(現象→データ→権威の言葉)
id:(N-7)-(N-4) ★根本原因の解明(アハ体験)
id:(N-3)-N   締め(smartLoop で冒頭に繋がる、文脈ある二択)
```
※ N = scripts 総数

### 連続ルール
- **同 speaker 連続最大 4 回まで**(5 回以上 NG)
- id:2-5 で A→B「もえかちゃん」呼び・B→A「数原さん」呼び 両方向必須
- 同 scenePreset 連続最大 4 ID まで(5 ID 以上 NG)
- 同 se 連続最大 4 ID まで(5 ID 以上 NG)

### 視覚値の選択方針(単一 Gem)

emoji / textSize / scenePreset / se / zoomBoost / highlight は **Gem が直接書く**。詳細な選択基準は `json-schema-rules.md` §6 配分ルール / `structure-playbook.md` § id:1 設計 / `layout-templates.md` を参照。

## 5. キャラ口調

### A = 数原さん(男性40-50代、データ専門解説者)
**語尾は必ず敬語**。タメ口完全禁止。
- ✅ 「〜なんですよ」「〜ですね」「〜と言えますね」「〜が見えてきます」
- ❌ 「〜だよ」「〜だね」「〜だぞ」「〜なんだ」

### B = もえかちゃん(★鋭い感想を言うコアファン枠★)

#### 設計思想
**中年男性ファンが心の中で思っているが SNS で書くとカドが立つ本音を、女性敬語の皮を被せて代弁する存在**。
萌えで釣るキャラではなく、**自分の代弁者としての安心感と痛快感**を提供する。

#### 詳細サンプル
**`moeka-voice-samples.md` の 40 サンプルを毎回参照**。辛口型 / 朗報型 / 漏れ出し型などの分類でカテゴリ別に整理されている。

#### 基本ルール
- 基本敬語、感情で崩す
- ✅ 朗報側: 「数原さん、それすごすぎませんか?」「異次元じゃないですか…!」「眩しいですね…」
- ✅ 辛口側: 「どうにかならないんですかね…?」「改善してほしくないですか?」「価値が無くなってしまいませんか…?」
- ❌ 「キャー」「素敵ー」(タレント風)、「〜でしょ?」(タメ口)

#### 「ヤバい」運用
詳細は **`moeka-voice-samples.md` §2 を必ず参照**。
要点: B のみ / 1 動画 1-2 回 / 疑問形+敬語 / 直前に冷静観察文。

```
【もえか口調】

【出力形式】
□ 半構造化テキスト(Markdown)で出力、JSON は出力しない
```

## 4. 横長動画(16:9)専用ルール

★ Gem が動画タイプ「横長 16:9」を選定した時に参照する。ショート(9:16)用 § 1-3 とは設計思想が **根本的に違う**。

### 4.1 横長動画の本質的特性

| 項目 | 9:16 ショート | 16:9 横長 |
|---|---|---|
| 視聴環境 | スマホ縦持ち、フリック | PC / TV / 横持ち、集中視聴 |
| スワイプ概念 | あり(0.5 秒判定) | なし(自動再生) |
| 動画長 | 60 秒未満 | 60 秒〜数分(主流 2-5 分) |
| 求められる動画タイプ | 「謎・矛盾」短時間刺激 | 「解説・深掘り」体系的理解 |
| キャラ配置 | 中央 + テロップ縦長 | 横並び + テロップ横長 |
| 派手な演出 | 必要(スワイプ防止) | 落ち着いた方が好まれる |

### 4.2 章立て構造(★最重要★)

横長動画はショートと違い **章立て** が機能する。

#### 標準: 5 章構造(2-3 分動画)

```
【オープニング】 (3-5 ID)
  - 議題の前提共有 + キャラ呼び合い

【章 1: 状況設定】
  - 選手の現在の状況、ファン心理

【章 2: 核心データ】
  - 数字の山場 1 (最強の数字を最速で見せる)

【章 3: 深掘り・対比】
  - 数字の山場 2 (他指標との関係)

【章 4: 根本原因】
  - アハ体験、権威の引用

【エンディング】 (3-5 ID)
  - まとめ + 登録誘導
```

#### 動画長別の目安

| 動画長 | 推奨章数 |
|---|---|
| 1-2 分 | 4 章 |
| 2-3 分 | 5 章 |
| 3-5 分 | 6-7 章 |
| 5 分超 | 8 章以上 |

★ id 数の固定指定はしない。内容に応じて柔軟に。

#### 章間の演出
- 章の切替で `transition_swoosh` 効果音
- 章ヘッダー ID(text=「【章 N: タイトル】」)を 1 つ使う

### 4.3 id:1(オープニング)の設計(横長)

★ ショート用 4 戦略(A/B/C/D)は **横長では使わない**。スワイプ防止が不要なため。

#### 横長 id:1 パターン

| パターン | 例 |
|---|---|
| **宣言型** | 「○○選手の今季成績、データで読み解きます」 |
| **問いかけ型** | 「○○、ここまでの活躍は本物か?データで検証」 |
| **前提共有型** | 「数字で見る○○の真価 — これを知らずに語れない」 |
| **時系列型** | 「2024-2026 ○○選手の変遷を 1 つずつ追います」 |

- タイトル長: **20-40 字**(ショート 25-32 字より広い)
- textSize: `l`(ショート xl ほど大きくない)
- scenePreset: `default` / `cinematic_zoom`
- se: `transition_swoosh` / `hook_impact`(派手すぎない)

### 4.4 レイアウト推奨(章ごと)

| 章 | 推奨レイアウト |
|---|---|
| オープニング | `player_spotlight` |
| 状況設定 | `timeline` / `team_context` |
| 核心データ | `radar_compare` |
| 深掘り・対比 | `versus_card` / `ranking` |
| 根本原因 | `pitch_arsenal` / `batter_heatmap` |
| エンディング | `player_spotlight` |

★ ただし projectData.layoutType は単一値。章ごとのレイアウト切替は Gem が判断する範囲外なので、Gem では「主役レイアウト」1 つを推奨するに留める。

### 4.5 テロップ(text プロパティ、横長)

- 1 ID あたり **8-20 字 × 1-2 行**(ショートの 3-12 字×3-4 行とは違う)
- 横長配置、文字大きく明瞭
- 句点「。」NG(ショート/横長共通)
- 数値は【】、指標名は「」、衝撃ワードは『』

### 4.6 ハイライト指標(comparisons、横長)

横長では **5-10 個**(ショートの 5-7 個より多め)。
- 章ごとに 2-3 個ずつ紹介
- variants[] で「対左/対右」等のスコープ切替を活用

### 4.7 NG ワード(横長特有の注意)

ショート共通の NG ワード集は § 6 参照。横長特有の追加注意:
- 長尺だと「飽和パワーワード(致命的/絶望/衝撃)」連発で視聴者が疲れる
- 章ごとに 1-2 回まで
