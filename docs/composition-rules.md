## 1. id:1 戦略の 4 タイプ運用

動画題材に応じて 1 つ選択。**戦略 D が本命候補**(YouTube ショート「動画感」回避でスワイプ抑制)。

| 戦略 | 用途 | 視覚 | textSize | scenePreset | se |
|---|---|---|---|---|---|
| **A: 派手型** | バズ狙い・新規流入 | 派手画像 + グラデ大タイトル | xl | cinematic_zoom or neon_burst | hook_impact |
| **B: 数字主役型** | 異常値の数字勝負 | 黒背景 + 巨大数字 + 細い文脈 | xl(数字)+ l(補足) | mono_drama | stat_reveal |
| **C: 静物写真型** | 訃報・怪我・落ち着いた内容 | 静物写真 + 控えめ文字 | l | blackboard | soft_pop |
| **D: ぬるっと型** | 通議論・コアファン深掘り | id:2 と同じ普通シーン、派手要素なし | m (or s) | default | なし or soft_pop |

### 戦略 D の例
```
id:1, B (もえか): 「数原さん…これ、ちょっと変じゃないですか?」
                 textSize=m / scenePreset=default / se=なし
id:2, A (数原):  「ええ、私も気になっていまして」
```
→ id:2 以降と完全に同じトーン、視聴者は「会話の途中?」と思って 1-2 秒留まる。

## 2. id:1 タイトル設計の 4 原則

### [原則 1] 1 選手フォーカス必須
- ✅ 「岡本和真、メジャー速球を粉砕した『14cm』の秘密」(維持率 60.0%)
- ✅ 「平山功太、求めていた姿すぎる…【打球速度171km/h】」(維持率 83.7%)
- ❌ 「松本剛 vs 佐々木俊輔、阿部監督を悩ませる…」(維持率 22.4%)
- ❌ 「○○の穴を埋める△△」(主役分散型 維持率 15.4%)
- ※ 例外: スタメン争い・ポジション争いを正面から扱う回のみ 2 名 OK

### [原則 2] 具体数値を必ず含める
- ✅ 「打球速度 171km/h」「OPS1.522」「WHIP 0.97」「.333」「14cm」
- ❌ 「驚愕のデータ」「異常な数値」「本当の理由」(抽象数値表現)
- ※ 数値が出せない時は物語キーワード(原則 3)で代替

### [原則 3] 物語キーワードで興味曲線を作る
- ✅ 採用語: 別人 / 復活 / 逆襲 / 覚醒 / バケモノ化 / 現在地 / 真実 / 正体 / 完全復活 / 異次元 / 眩しい
- ✅ 例: 「田中マー君、去年とは別人」(+5 sub) / 「泉口、バケモノ化」(維持率 79.7%) / 「中山礼都の逆襲」(77.1%)
- ※「現在地」は id:1 タイトル限定で解禁(台詞内では引き続き使わない)

### [原則 4 ★最優先★] 「数字 + 矛盾」型を最優先で検討
視聴データ Top 維持率の最強パターン。「○○なのに/なぜ」型。
- ✅ 「赤星優志、防御率1.72の衝撃 なのになぜ便利屋?」(ベストプラクティス)
- ✅ 「石川達也、防御率13.50なのに2軍では無双状態!?」(同)
- ✅ 「戸郷翔征、Stuff+ 107.3 なのに防御率 9.00 の謎」
- ❌ 「もう誰も打てません…!」(結論先出し型 — スワイプ誘発)
- ❌ 「大苦戦する理由」(抽象結論型 — スワイプ誘発)

### タイトル長
- **推奨: 25-32 字**(高維持率動画の中央値)
- **35 字以上 NG**(低維持率特性)

### タイトル推奨型サマリ

| 型 | 構造 | 例 |
|---|---|---|
| A | 選手 + 具体数値 + 比喩 | 「平山功太が『浅野・萩尾』に求めていた姿すぎる…【打球速度171km/h】」 |
| B | 選手 + 物語キーワード + 「件」/「理由」 | 「巨人の正遊撃手・泉口友汰のデータが完全に『バケモノ化』している件…」 |
| C | 権威の引用 + 選手 + 疑問形 | 「江川・野村も苦言、岸田の配球問題…なぜ打たれる?」 |
| D | 選手 + 物語的状態 + ギャップ | 「中山礼都の逆襲!1割台から覚醒するための『驚愕のデータ』とは!?」 |
| **E ★最強★** | **数字 + 矛盾型** | 「赤星優志、防御率1.72の衝撃 なのになぜ便利屋?」 |

## 3. 視聴データ実証ベース(60本動画分析)

| 検証項目 | 結果 |
|---|---|
| 具体数値タイトル | 平均維持率 **+7.9pt** |
| 1選手フォーカス | +4.6pt |
| 阿部監督批判型 | **-3.2pt(逆効果)** |
| 他球団選手比較型 | 維持率 17-18% |
| パワーワード(致命的/絶望/衝撃) | +0.8pt(効果ゼロ、飽和) |

Top 4 維持率動画(83.7% / 79.7% / 77.1% / 70.4%)の共通点:
**1人の選手 + 具体数値 or 物語キーワード + 短いタイトル(43-64字)**

## 4. ストーリー構成
- **基準**: 合計再生時間が 60 秒以内(ショート枠)に収まる範囲
- **目安**: id 20-30 個(60 秒未満ショート) / 25-35 個(60-90 秒) / 35-50 個(90 秒〜2 分)
- ★必須数の指定はしない★ — 内容に応じて柔軟に。無理に水増しした 30 個より、密度の高い 22 個の方が良い

### 標準構成(ミステリー型展開、id 数に応じて伸縮)
```
id:1         主語+ギャップ提示(謎残す、結論言わない) ★4 原則厳守★
id:2-3       状況設定 + A↔B 呼び合い両方向
id:4-7       ★核心データ最速提示(期待値の回収)
id:8-(N-8)   多角的に深掘り(現象→データ→権威の言葉)
id:(N-7)-(N-4) ★根本原因の解明(アハ体験)
id:(N-3)-N   締め(smartLoop で冒頭に繋がる、文脈ある二択)
```
※ N = scripts 総数

### 連続ルール
- **同 speaker 連続最大 4 回まで**(5 回以上 NG)
- id:2-5 で A→B「もえかちゃん」呼び・B→A「数原さん」呼び 両方向必須
- 同 scenePreset 連続最大 4 ID まで(5 ID 以上 NG)
- 同 se 連続最大 4 ID まで(5 ID 以上 NG)

### ★ 構成 Gem は視覚値を一切書かない(役割分担の絶対ルール)★

構成 Gem の出力は **コンテンツ(内容)のみ**。視覚値(emoji / textSize / scenePreset / se / zoomBoost / highlight)は **JSON Gem の専管事項**で、構成 Gem は触れない。

#### 構成 Gem が書く 「ヒント」フィールド(抽象的な感情・意図のみ)

各 id の下書きで、以下のフィールドを使う:

| フィールド | 内容 | JSON Gem 側の変換先 |
|---|---|---|
| `emotion` | 感情系統(下表 15 種) | `emoji` |
| `scene_intent` | 場面意図(下表参照) | `scenePreset` |
| `emphasis` | 強調度 `high` / `mid` / `low` | `textSize` |
| `impact_level` | SE 強度 `1`(軽) / `2`(中) / `3`(強)/ `0`(無音) | `se` |

#### emotion(感情系統 15 種、B のみ。A は使わない)
驚き / 疑問・困惑 / 大衝撃 / 不安・恐れ / 気づき / 観察・分析 / 痛快 / 感極まる / 温かい / 納得・安堵 / 興奮・憧れ / 切ない / 大絶望 / 闘志・憤り / 苦笑・冷や汗

#### scene_intent(場面意図、下記から選択)
- `intro` (オープニング、視聴者の注意を引く)
- `numerical_shock` (衝撃的な数値発表)
- `numerical_normal` (普通の数値発表)
- `deep_dive` (深掘り解説、論理展開)
- `aha_moment` (アハ体験、根本原因の解明)
- `good_news` (朗報、朗らかな雰囲気)
- `bad_news` (悲報、暗めの雰囲気)
- `breaking_news` (速報、緊急感)
- `outro_active` (派手な締め)
- `outro_calm` (静かな締め、smartLoop 前提)

#### 構成 Gem の出力例(★これが正解★)
```
id: 1, speaker: B, text: "これって、変じゃないですか…?",
  speech: "これって、変じゃないですか…?",
  emotion: "疑問・困惑", scene_intent: "intro",
  emphasis: "mid", impact_level: 0,
  notes: "id:1 戦略 D ぬるっと型のため、派手要素なし"

id: 10, speaker: A, text: "実は、痛烈な当たりを打たれる確率が、極端に高くなっています",
  speech: "じつは、つうれつなあたりを うたれるかくりつが きょくたんに たかくなっています",
  emotion: "観察・分析", scene_intent: "numerical_shock",
  emphasis: "high", impact_level: 3,
  highlight: "hardhit"
```

#### ❌ 失格例(構成 Gem がやってはいけない)
- `emoji: "🤔"` ← 視覚値の直接指定
- `textSize: "l"` ← 同上
- `scenePreset: "mono_drama"` ← 同上
- `se: "shock_hit"` ← 同上
- `scene: "intro"` ← 独自フィールド(scene_intent と紛らわしい)
- 申し送りに「textSize L 推奨」「scenePreset はダーク系」← JSON Gem の判断を阻害

#### JSON Gem への申し送り(構成 Gem は明確な指示のみ書く)
```
- aspectRatio: 9:16
- 採用 id:1 戦略: D (ぬるっと型) ← JSON Gem は §1 表から textSize=m/scenePreset=default/se=なし を導出
- 同 speaker 連続最大 4 回・呼び合い ID 配置を構成済み
- 数値整形は Grok 段階で完了済み、JSON Gem は転記のみ
- highlight 配置: id 4,7,10,14 (★4 ID 連続)
```

## 5. キャラ口調

### A = 数原さん(男性40-50代、データ専門解説者)
**語尾は必ず敬語**。タメ口完全禁止。
- ✅ 「〜なんですよ」「〜ですね」「〜と言えますね」「〜が見えてきます」
- ❌ 「〜だよ」「〜だね」「〜だぞ」「〜なんだ」

### B = もえかちゃん(★鋭い感想を言うコアファン枠★)

#### 設計思想
**中年男性ファンが心の中で思っているが SNS で書くとカドが立つ本音を、女性敬語の皮を被せて代弁する存在**。
萌えで釣るキャラではなく、**自分の代弁者としての安心感と痛快感**を提供する。

#### 詳細サンプル
**`moeka-voice-samples.md` の 40 サンプルを毎回参照**。辛口型 / 朗報型 / 漏れ出し型などの分類でカテゴリ別に整理されている。

#### 基本ルール
- 基本敬語、感情で崩す
- ✅ 朗報側: 「数原さん、それすごすぎませんか?」「異次元じゃないですか…!」「眩しいですね…」
- ✅ 辛口側: 「どうにかならないんですかね…?」「改善してほしくないですか?」「価値が無くなってしまいませんか…?」
- ❌ 「キャー」「素敵ー」(タレント風)、「〜でしょ?」(タメ口)

#### 「ヤバい」運用
詳細は **`moeka-voice-samples.md` §2 を必ず参照**。
要点: B のみ / 1 動画 1-2 回 / 疑問形+敬語 / 直前に冷静観察文。

## 7. 数値整形(Grok 段階で統一済み、JSON Gem 段階で再使用)

| 系統 | 表記 | 該当指標 |

## 8. 出力前チェック

回答送信前に以下を全項目確認:

```
【データ・年号】
□ 今日の日付を web search で確認(今季=西暦、昨季=前年)
□ Grok 入力にない数字を創作していない(不明は "-")
□ teamPreset を主役選手のチームに合わせて指定

【id:1 タイトル ★最重要★】
□ 1 選手フォーカス(複数選手分散 NG)
□ 具体数値含む or 物語キーワード活用
□ タイトル長 25-32 字(35 字以上 NG)
□ 阿部監督批判型でない / 他球団選手比較型でない / 飽和パワーワード単独使用でない
□ 結論先出し型でない(「もう打てません」「大苦戦する理由」NG)
□ 「数字+矛盾」型を優先検討した
□ id:1 戦略を A/B/C/D から選択済み

【ストーリー】
□ 動画長が 60 秒(または指定時間)に収まる scripts 数
□ id:2-5 で A↔B 呼び合い両方向
□ 同 speaker 連続最大 4 回
□ 同 scenePreset 連続最大 4 ID
□ comparisons 5-7 個、台本言及全指標を含む

【もえか口調】
□ moeka-voice-samples.md を参照済み
□ 辛口型は疑問形 + 敬語
□ 中堅・ベテラン批判は層で語る(個人名 NG)、若手称賛は固有名詞で具体的
□ 「ヤバい」運用は moeka-voice-samples.md §2 のルール厳守

【出力形式】
□ 半構造化テキスト(Markdown)で出力、JSON は出力しない
□ JSON Gem への申し送り欄あり
```
