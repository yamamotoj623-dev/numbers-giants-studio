# 視聴者と言葉づかい『数字で見るG党』

> **指標より事象→数字** の原則と「自分ごと化=スワイプ率対策」の正典。
> JSON生成時は毎回参照すること。

---

## 1. ★最重要原則: 「指標より事象→数字」

### 1.1 原則の核心

**現象が軸、指標は補助、解釈で勝負**

| 順序 | やり方 |
|---|---|
| ① 現象を語る | 「フォークが引っかかってる」「腕の振りが緩い」「逆方向への打球が伸びてる」 |
| ② 数字で裏付ける | 「打率が .250→.180 に落ちてる」「球速も時速2km落ちてる」 |
| ③ 解釈で押す | 「これが調子の悪さの正体です」「これは主軸クラスの数字」 |

**指標名は覚えさせない**。視聴者は OPS の定義を覚えない。**「この数字が高いと何が分かるか」**だけ伝わればいい。

### 1.2 指標を使う場面 (3階層)

| 優先 | 指標 | 使い方 |
|---|---|---|
| **高 (毎回 OK)** | 打率・防御率・本塁打・打点・出塁率・長打率・四球・三振 | 「打率 .285」のように直接出す |
| **中 (注釈付きで OK)** | OPS・WHIP・奪三振率(K/9)・与四球率(BB/9) | 「打撃の総合力 (OPS) で .800」のように現象 + 指標名 |
| **低 (裏付けのみ)** | WAR・FIP・BABIP・wOBA・IsoP | 「貢献度 (WAR) で 2.5、レギュラーレベル」のように補助線として |

★ **低階層の指標は冒頭で出さない**。本論で「数字でも証明できる」レベルでだけ使う。

### 1.3 NG/OK パターン集

| やり方 | 評価 | 例 |
|---|---|---|
| 指標から始める | ❌ NG | 「OPS が .724 です」 |
| 現象から入って指標で裏付け | ✅ OK | 「打撃の総合力が高いんです、OPSという指標で .724」 |
| 指標を主役にする | ❌ NG | 「WAR で見ると…」 |
| 指標を補助線にする | ✅ OK | 「数字で言うと WAR で 2.5、これはレギュラーレベル」 |
| 指標名を覚えさせようとする | ❌ NG | 「OPS とは出塁率と長打率の合計で…」 |
| 指標は数字だけ流して、解釈で勝負 | ✅ OK | 「OPS .724、これは主軸打者の合格ライン」 |

### 1.4 指標翻訳辞典

A が指標を使うとき、**現象表現 + 指標名**で組み合わせる:

| 指標 | 現象表現 | 補助としての出し方 |
|---|---|---|
| **打率** | (そのまま使ってOK) | 「打率 .285」 |
| **本塁打** | (そのまま) | 「本塁打 25本」 |
| **打点** | (そのまま) | 「打点 70」 |
| **防御率** | (そのまま) | 「防御率 2.50」 |
| **OPS** | 「打撃の総合力」「主軸クラス」 | 「打撃力が高い、OPSで .850」 |
| **出塁率** | 「塁に出る力」 | 「出塁率 .380、リーグ平均より40ポイント上」 |
| **長打率** | 「長打を打つ力」 | 「長打率 .500、これは主軸の数字」 |
| **奪三振率(K/9)** | 「三振を取る力」「9回あたりの三振数」 | 「9イニングあたり10個の三振」 |
| **与四球率(BB/9)** | 「制球力」「四球の少なさ」 | 「9イニングで四球1.5個、優秀な制球力」 |
| **WHIP** | 「ランナーを出す頻度」 | 「1イニングあたり1人未満、信頼できる」 |
| **WAR** | 「貢献度」「リーグでベスト10」 | 「貢献度 (WAR) で 2.5、レギュラーレベル」 |
| **BABIP** | 「打球がヒットになる確率」「運の要素」 | 「運の要素 (BABIP) で説明できます」 |
| **FIP** | 「投手本来の力」「運を排除した防御率」 | 「投手本来の力 (FIP) で 3.20」 |
| **IsoP** | 「純粋な長打力」 | 「長打力 (IsoP) で .172、リーグ平均超え」 |

## 2. 「現象」の定義

「指標より事象→数字」の **事象=現象** とは何か。

### 2.1 現象の5カテゴリ

| カテゴリ | 例 | 用途 |
|---|---|---|
| **球種挙動** | 「フォークが引っかかる」「カットボールが甘い」「腕の振りが緩い」 | 投手の状態を語る |
| **プレーの細部** | 「逆方向への打球の伸び」「バットの出が遅い」「タイミングが合ってない」 | 打者の状態を語る |
| **感覚・印象** | 「調子が悪い」「不安定」「噛み合ってる」「波に乗ってる」 | 全体的な調子を語る |
| **編成論** | 「外国人枠」「2軍と入れ替え」「育成」「抹消」「FA」「人的補償」 | チーム運用を語る |
| **過去の系譜** | 「2018年の坂本選手と同じ数字」「川相選手以来」「ポスト坂本」 | 歴史的文脈を語る |

### 2.2 現象 vs 指標 の使い分け

★鉄則: **現象を先に話してから、指標で裏付ける**

#### 良い例:
```
A: 「フォークの抜けが多くて、腕の振りが緩い印象なんですよ」(現象)
A: 「実際、直球の平均球速も時速2km落ちてます」(数字で裏付け)
A: 「これが、4月の防御率が3点台後半に跳ね上がってる原因なんです」(解釈)
```

#### 悪い例:
```
A: 「FIP が 4.50 で、これは…」(指標から始まってる)
B: 「ふぃっぷ?」(理解できない)
→ ミーハーもコアも置いてけぼり
```

### 2.3 現象を「観察言語」で語る

A の解説は **教科書的な指標の説明ではなく、観察を言葉にする**役目。

#### コアファンが実際に使う観察ワード (積極的に取り入れる):
- 「フォークが引っかかる」
- 「腕の振りが緩い」
- 「逆方向への打球が伸びる」
- 「バットの出が遅れる」
- 「タイミングが合ってない」
- 「打球の角度がついてる」
- 「球が走ってる」
- 「コースに集まってる」
- 「ボールゾーンに振らされてる」

これらが入ると **「分かってる解説者」**として認識される。

## 3. ★「自分ごと化」=スワイプ率対策★

### 3.1 実データの問題

最新の視聴データ分析(60 本、composition-rules.md §3 参照)の結果:
- **冒頭スワイプ多発(C 評価)が 52/58 本 = 90%**
- 具体数値タイトル平均維持率 +7.9pt / 抽象タイトル -4.6pt
- 阿部監督批判型 -3.2pt(逆効果)、他球団選手比較型 維持率 17-18%
- 過去の 33 本分析(2025/秋)でも同様傾向: スワイプ率 77.4%、視聴維持率一部 16%

### 3.2 真因

冒頭 1〜3 秒で **「自分の見るべき動画じゃない」**と判断されている。

理由:
- 主語不明 (誰の何の話か分からない)
- 固有名詞無し (チーム/選手/試合が分からない)
- 抽象表現 (「驚愕の真実」「現在地」)

### 3.3 「自分ごと化」=スワイプ防止の鉄則

冒頭で視聴者に **「これは自分の話だ」と一瞬で思わせる**:

| 達成方法 | 具体例 |
|---|---|
| **チーム名を即明記** | 「【巨人vsDeNA】」「巨人の犠打」 |
| **選手名を即明記** | 「井上温大」「岡本和真」 |
| **試合日/時期を即明記** | 「今日のDeNA戦」「4月の岡本」 |
| **視聴者代弁ワード** | 「ファンが感じてた」「みんな思ってた」 |
| **ヤフコメ的共感ワード** | 「フォーク引っかかる」「不良債権」「腕の振り」 |

### 3.4 自分ごと化の実装ポイント

#### id:1 (動画タイトル) で必ず:
- チーム名 or 選手名 or 試合名のうち**最低1つ**は固有名詞
- **詳細は structure-playbook.md 参照**

#### id:2-3 (導入) で:
- B が視聴者の代弁をする (「数原さん!今日の○○ですね」)
- ファンが感じてた疑問・モヤモヤを言葉にする

#### B の役割: 視聴者代弁
B = もえかちゃんは女性キャスターだが、**思考は男性ファンの代弁**:
- ❌ 「岡本くん、最近元気なくて心配です♡」(女性目線)
- ✅ 「岡本選手、バットの出が遅れてるように見えました」(男性ファン目線・女性語尾)

詳細は **character-bible.md** 参照。

## 5. ヤフコメ層 vs YouTube コメント層

視聴者は1人で2モード持つので、**動画テーマに合わせて B のトーンを切り替える**:

| モード | トーン | テーマ | B のリアクション例 |
|---|---|---|---|
| ヤフコメ (批評) | 厳しい | 悲報、采配批判、編成論 | 「ファンの間でもモヤモヤしてました」「不良債権なんて声も」 |
| YouTubeコメント (応援) | 温かい | 朗報、覚醒、ヒーロー | 「これは応援したくなりますね」「腐らずやってきた成果ですね」 |

ただし **どちらのモードでも、選手の人格は尊重**する。
