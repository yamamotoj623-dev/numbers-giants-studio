# レイアウト方向性『数字で見るG党』 v2

> このドキュメントは、各レイアウトが**動画の中で何を担うか**を定義する正典です。
> Gemini Custom Gem / Grok カスタムエージェントの **Knowledge File** として参照され、レイアウト選択の判断基準になります。
> 同時に、レイアウト改修 (コード変更) の北極星でもあります。

---

## 0. このドキュメントの目的

レイアウトの個性は **動画60秒のどの局面で・何を担うか** で決まります。
これを明確にすることで:

- AI (Gemini / Grok) のレイアウト選択精度が上がる
- レイアウトごとに「なぜこのデザインか」の根拠が生まれる
- 視聴者が「次にこういうレイアウトが来るな」と予感できる

## 2. ストーリーアーク上のレイアウト位置

### 60秒のショート動画 標準アーク

```
時間  │ 役割              │ 視聴者の心理              │ 主に使うレイアウト
──────┼──────────────────┼─────────────────────────┼─────────────────────
0-3秒 │ フック             │ 「これ自分の動画?」      │ (フック画面)
3-15秒│ 問題提起・状況説明  │ 「何が起きてるんだ?」    │ ranking / team_context / radar_compare
15-30秒│ 深掘り1 (なぜ)    │ 「もっと知りたい」       │ timeline / player_spotlight
30-45秒│ 深掘り2 (角度変え)│ 「他にもあるのか」       │ versus_card / pitch_arsenal / batter_heatmap
45-55秒│ 結論・転換         │ 「それで?」              │ versus_card / radar_compare
55-60秒│ 締め (二択疑問)    │ 「自分はどう思う?」      │ (アウトロ画面)
```

### 各レイアウトのアーク上の主戦場

```
0秒 ─────────────────────────────────────────── 60秒

[ranking]      ━━━━━━━━━━━━━━━━━           序盤の衝撃で掴む
[team_context] ━━━━━━━━━━━━━                全体像の提示で導入
[radar_compare] ━━━━━━━━━━━━━━━━━━━━━━━━     汎用、序盤〜終盤
[timeline]                ━━━━━━━━━━━━━━     中盤で「何が起きたか」
[player_spotlight]               ━━━━━━━━━   中盤で個人深掘り
[pitch_arsenal]                  ━━━━━━━     投手の手の内 (中盤)
[batter_heatmap]                 ━━━━━━━     打者の傾向 (中盤)
[versus_card]                          ━━━━━ 終盤で結論強化
```

## 3. 共通設計ルール

これらは**全レイアウト共通**で守る:

### 3.1 テロップ被り防止
- 全レイアウトで**下部 32% (画面下から約205px)** はテロップ領域として侵食しない
- レイアウトの主要要素 (グラフ、数値、選手名) はこの領域より**上に配置**
- やむを得ず下部まで広がる場合は**テロップ表示時に縮小・透明度調整**

### 3.2 テロップ枠の背景透明度
- テロップ枠の背景は**完全不透明にしない**
- レイアウトが透けて見える程度の透明度 (0.85-0.9 程度) で:
  - レイアウトと一体感が出る
  - 隠したい情報が完全に消えない (情報量を保持)

### 3.3 デザイン言語の統一
- **オレンジ系**: 巨人カラー、注目・好調・主役 (mainPlayer)
- **金色**: ベスト・勝者・1位
- **赤色**: ワースト・警告・敗者
- **青色 (sky)**: 比較対象・sub・敵チーム
- **グレー**: 中性・平均・互角

### 3.4 アニメ速度
- フェード切替: 0.45秒
- 主役要素の出現: 0.6秒 (タメ感)
- 強調パルス: 1.5秒周期

### 3.5 注目強調の禁則
- **同時刻に「◀注目」マークが付くのは1人だけ**
- 複数選手をフォーカスしたい場合は、**動画の異なるシーンで時間差**で focusEntry を切り替える
- ranking で全員 isMainPlayer:true 等は禁止 (注目の意味が消える)

## 4. 各レイアウトの役割と方向性

### 4.1 radar_compare (レーダー比較) — 役割: 「全体像」

**ストーリーテリング上の役割**: 全体像を提示する標準形

**メッセージ**: 「総合的にどうなのか」「どこが優れてどこが弱いのか」

**アーク上の位置**: 序盤〜終盤 (汎用)

**個性の方向**: **標準・汎用の導入役**。派手さより**情報の整理感**で勝負。

**改修方針**: **現状維持**。

**典型的な使い場面**:
- 動画の序盤で「総合的な状態」を提示
- 比較動画 (今季 vs 昨季、A選手 vs B選手) の標準形
- 終盤の振り返りで再登場することも

---

### 4.2 timeline (時系列) — 役割: 「変化」をドラマチックに見せる

**ストーリーテリング上の役割**: 「**ここで何が起きた**」を語る

**メッセージ**: 「○月から急変した」「この時点で覚醒/崩壊した」

**アーク上の位置**: **中盤** (深掘り1の主役)

**個性の方向**: **転換点の演出**。ドラマ性を持たせる。

**改修方針**:

#### 軸の柔軟化 (重要)
時間単位を以下に拡張:
- `unit: "day"` — **日別** (シーズン直近、データ薄い時期に最適)
- `unit: "week"` — 週別 (1ヶ月以内の変化を見せる)
- `unit: "month"` — 月別 (シーズン中の長期変化、現状デフォルト)
- `unit: "year"` — **年別** (チーム主語で複数シーズンの推移)

軸単位によって points 配列の意味と推奨点数が変わる:

| unit | 推奨点数 | 想定シーン |
|---|---|---|
| day | 5-15 | 開幕直後、直近1週間の変化 |
| week | 4-12 | 1〜3ヶ月の変化 |
| month | 3-7 | シーズン通年の月別推移 |
| year | 3-10 | チームの複数年推移 (チーム動画) |

#### シンプル化
- ❌ 現状: main + sub の比較線、目盛り、凡例、月別/週別ラベル
- ✓ 改修: **必要最小限の表示**
  - sub (比較線) は**任意化** (1選手の推移だけ見せたい場合に対応)
  - データ最低 **2点でOK** (現状3点要件を緩和)
  - 凡例は小さく、目盛りは控えめに

#### ドラマ性を加える
- **転換点ハイライト**: highlight=true のポイントは:
  - 線が**急激に太く**なる (上昇なら太く・明るく、下降なら太く・暗く)
  - 点が**脈動** (パルスアニメ)
  - 数値ラベルが**拡大表示**
- **上昇/下降で色が変わる**:
  - 上昇 (覚醒) → オレンジ → ゴールド系
  - 下降 (崩壊) → グレー → 赤系
- **「ジャンプ」演出**: 大きな変化があったポイントで線が**段差的**に見える

#### データスキーマの改修
```
旧: { 
  unit: "month"|"week",
  metric: "...",
  points: [{label, main, sub, highlight}, ...] 
}
新: { 
  unit: "day"|"week"|"month"|"year",  // year を追加
  metric: "...",
  points: [{label, value, highlight, isPeak, isBottom}, ...],
  compareLine?: [{label, value}, ...]   // sub を任意化
}
```

#### 互換性
- 旧 `points[].main` は内部で `value` に変換 (旧データを動かすため)
- 旧 `points[].sub` は `compareLine` として変換

**典型的な使い場面**:
- 朗報・覚醒: 「4月から打率が急上昇」(unit: week)
- 悲報・不振: 「7月から防御率が崩壊」(unit: week)
- チームの長期推移: 「2020年から得点力推移」(unit: year)
- 開幕直後の変化: 「4月の打率推移」(unit: day)

**避けるべき場面**:
- データが2点未満
- 変化のない選手 (timeline 出しても意味なし)

### 4.3 versus_card (対決カード) — 役割: 「対決感」で結論を強める

**ストーリーテリング上の役割**: 1対1の**対比で結論を確定**させる

**メッセージ**: 「○ vs △、勝者はこっち」 OR 「○ vs △、こんなに負けてる」

**アーク上の位置**: **中盤〜終盤**

**個性の方向**: **対決感**。勝者だけでなく、**敗者強調**にも対応。

**改修方針 (重要)**:

#### mood 切替の追加 (最重要)
```
旧: 勝者を自動判定して光らせる
新: mood フィールドで強調対象を制御
```

mood の3パターン:

| mood | 用途 | 演出 |
|---|---|---|
| `"main_wins"` | mainPlayer の勝利を強調 | mainPlayer 側に WIN・光のオーラ |
| `"main_loses"` | mainPlayer の敗北を強調 (差を見せたい動画) | mainPlayer 側に「**○○差**」の警告マーク、subPlayer 側に "REF" (基準) マーク |
| `"close"` | 互角を見せる | 中央に "DRAW" or "互角" マーク、両側ともグレー寄り |

「main_loses」を使う場面例:
- 「岡本がいない巨人打線 vs リーグ平均」(mainPlayer=巨人、巨人が負けてることを強調)
- 「○○選手 vs 同年齢のレジェンド」(差を見せて課題を提示)

#### 対決感の強化
- **中央の「VS」を主役に**: 火炎・雷風の演出
- **左右のコントラスト**: 強調側に光、もう片方は影
- **WIN マーク**: mood:"main_wins" のとき、勝者側に**チャンピオンベルト風**
- **差の表示**: 「**差●**」を中央 VS の下に表示

#### スポットライトの方針
- 視聴者から見て「main_wins の場合」も「main_loses の場合」も、**スポットライトは強調したい側**
- mainPlayer に必ずスポットを当てるのではなく、**mood に応じて**強調を切替

#### データスキーマの改修
```
旧: { 
  overall: {main, sub},
  categoryScores: [{label, kana, main, sub, rawMain, rawSub}]
}
新: 上記 + mood: "main_wins"|"main_loses"|"close" (省略時は自動判定で main_wins / main_loses)
```

**典型的な使い場面**:
- 1対1の対決動画 (増田陸 vs 浦田)
- 巨人 vs 阪神 のような**チーム対決**
- 「数字でこんなに負けてる」を強調 (mood: main_loses)

**避けるべき場面**:
- 3人以上の比較 (それは ranking)
- 「対決」じゃなく「並列」のとき

### 4.4 team_context (チーム文脈) — 役割: 「チームの表情」を見せる

**ストーリーテリング上の役割**: チーム全体の状態を**1枚絵**で伝える

**メッセージ**: 「チームとして強い・弱い」 OR 「巨人 vs セ平均でこう」

**アーク上の位置**: **序盤〜中盤**

**個性の方向**: **チームカラー** + **強さ・弱さが一目でわかる**

**改修方針 (重要)**:

#### モード分岐 (新)
team_context は**2つのモード**で動かす:

##### モード A: チームビュー (`mode: "single"`)
チーム単体の状態を見せる。3要素ブロック構造。

```
┌─────────────────────────────┐
│ 【打線】 (赤・好調 / 青・不調)  │
│  打率: .268 (5位) ●●●●○   │
│  本塁打: 32本 (4位) ●●●○○ │
│  得点: 178 (3位) ●●●●○   │
├─────────────────────────────┤
│ 【投手】(青・好調 / 赤・不調)  │
│  防御率: 2.85 (1位) ●●●●●  │
│  奪三振: 412 (2位) ●●●●○  │
│  失点: 145 (1位) ●●●●●    │
├─────────────────────────────┤
│ 【采配】                       │
│  代走起用率: 高              │
│  バント傾向: 多              │
│  継投の早さ: 早              │
└─────────────────────────────┘
```

各ブロックは**5段階の◯●**で強弱表示。

##### モード B: チーム間比較ビュー (`mode: "compare"`)
巨人 vs セ平均、巨人 vs 上位3チーム平均、などの**1対多比較**。

```
┌─────────────────────────────────────┐
│ 巨人 vs セ平均                          │
├─────────────────────────────────────┤
│ 打率                                   │
│  .268 (巨人) ━━━━━━━━━━━━━━━━━━━━     │
│  .268 (セ平均) ━━━━━━━━━━━━━━━━━━━━   │  互角
├─────────────────────────────────────┤
│ 本塁打                                 │
│  32本 (巨人) ━━━━━━━━━━              │
│  41本 (セ平均) ━━━━━━━━━━━━━━━━━━     │ ▼ 9本不足
└─────────────────────────────────────┘
```

→ **チーム間の差分が一目**でわかる。
→ 巨人 vs 阪神のような**1対1チーム対決**は **versus_card** に任せる。team_context は**「巨人とリーグの関係性」**を見せるレイアウト。

#### スタジアム背景
- 上部にスタジアムのシルエット (うすく、雰囲気を作る)
- 全体トーンを**オレンジ系**で統一 (巨人カラー)

#### 提案オーダー機能の扱い
- **削除** (使い所が限定的、データ生成も難しい)

#### データスキーマの改修
```
旧: { lineup: [...], roles: [...] }
新: 
  // モード A
  { 
    mode: "single",
    batting: { stats: [{label, value, rank, score(1-5)}, ...], note? },
    pitching: { stats: [{label, value, rank, score(1-5)}, ...], note? },
    management: { traits: [{label, level: "高"|"中"|"低"}, ...], note? }
  }
  // モード B
  | { 
    mode: "compare",
    target: "セ平均" | "上位3チーム平均" | "首位",
    stats: [{label, mainValue, compareValue, diff, mainBetter: bool}, ...]
  }
```

**典型的な使い場面**:
- モードA: チーム不振動画の冒頭で「打線が4位、投手1位だから投手任せ」
- モードA: 編成論動画の冒頭で「全体像を踏まえて編成を語る」
- モードB: 「巨人 vs セ平均」(リーグ内のポジション)
- モードB: 「巨人 vs 上位3チーム平均」(優勝に必要な差)

**避けるべき場面**:
- 個別選手の動画 (player_spotlight が適切)
- 巨人 vs 阪神のような1対1 (versus_card が適切)
- セリーグ内6球団順位 (ranking が適切)

### 4.5 ranking (ランキング) — 役割: 「衝撃」を与える

**ストーリーテリング上の役割**: 序盤で**強烈な衝撃**を与えて視聴者を掴む

**メッセージ**: 「実は最下位」「実はトップ」「○○がワースト/ベスト」

**アーク上の位置**: **序盤** (フックの直後、3-15秒の主役)

**個性の方向**: **衝撃感・序列ジャンプ感**。トーンの差・注目選手の脈動で「これはやばい」と感じさせる。

**改修方針 (重要)**:

#### mood 切替 (ベスト系 / ワースト系)

| mood | 用途 | トーン |
|---|---|---|
| `"best"` | 上位の活躍を見せる | 金・明るい、王冠、上昇感 |
| `"worst"` | 下位の不振を見せる | 赤・暗い、警告、下降感 |
| `"neutral"` | 通常の順位表 | 中性 |

##### mood: "best" のとき
- 1位は**金色の王冠 + 光のオーラ**
- 行が**上から下に降ってくる**ような出現アニメ
- ベスト3に**金・銀・銅メダル**

##### mood: "worst" のとき
- ワースト1位は**赤い警告アイコン + 暗い背景**
- 行が**下から沈んでいく**演出
- ワースト3に**▼マーク** (下降を示す)

#### 注目選手の演出強化 (focusEntry)
- focusEntry 指定された行は:
  - **1.04倍に拡大**
  - **脈動アニメ** (光が脈打つ)
  - 文字が**ゴールドに発光** (best) または **赤く発光** (worst)
  - 「**◀ 注目**」マークが**振動**
- **同時刻に強調されるのは1人だけ**: 全員 isMainPlayer:true は禁止 (3.5節参照)

#### 順位の動き感
- 行が**順番に出現** (上から下に、0.1秒ずつずれて)
- focusEntry の行は**最後に**強調出現 (タメから一気に出る)

#### 圏外マーカーの強化
- 注目選手が10位圏外なら:
  - 現状の「⋯」を**「圏外」と明示**
  - 「**12位**」みたいに具体順位を併記

#### チーム単位でのランキング対応 (新)
ranking の entries は選手だけでなく**チーム**もエントリにできる:

```
{ rank: 1, name: "阪神",   value: ".285", sub: "32本", isMainPlayer: false },
{ rank: 2, name: "DeNA",   value: ".272", sub: "28本", isMainPlayer: false },
{ rank: 3, name: "巨人",   value: ".268", sub: "26本", isMainPlayer: true },  // ← 巨人を強調
...
```

→ セリーグ内6球団の順位、12球団順位等に対応。
→ entry.isMainPlayer:true で巨人を**自動強調**。
→ 「シルエット」表示はチーム時はチームロゴ的なシンプル表示でもよい (将来検討)。

#### データスキーマの追加
```
旧: { mode: "single"|"multi", metrics: [{id, label, kana, unit, entries: [...]}] }
新: 上記 + mood: "best"|"worst"|"neutral", showCutoff?: bool
   entries: [{rank, name, value, sub?, isMainPlayer?, isTeam?}]
   isTeam: true のとき、チーム表示として扱う
```

**典型的な使い場面**:
- ワースト動画の序盤 (5人ワースト → 各人を spotlight で深掘り)
- ベスト動画の序盤 (リーグ上位 5人を見せる)
- セリーグ内チーム順位
- 12球団チーム順位

**避けるべき場面**:
- 1選手のみの動画 (player_spotlight が適切)
- 比較が3人以下 (versus_card or それより)
- 1対1チーム比較 (versus_card)
- 巨人 vs リーグ平均 (team_context モードB)

---

### 4.6 player_spotlight (選手スポット) — 役割: 「主役感」で深掘りする

**ストーリーテリング上の役割**: 1選手の**世界観**を見せる、深掘り中の主役

**メッセージ**: 「この選手はこういう存在」「数字でこう語れる」

**アーク上の位置**: **中盤** (深掘り1 or 2 の主役)

**個性の方向**: **ポートレート感・主役感**。雑誌の特集記事のように、選手単体を**美しく見せる**。

**改修方針 (重要)**:

#### ビジュアルの方向性
- **シルエットを劇画タッチで巨大に**: 現状より大きく、画面の主要部分を占める
- **名前カード**: シルエット下に**サインカード風**の選手名・番号・背番号
- **プライマリ指標を画面中央に巨大に**: フォントサイズを最大級
- **サブ指標は周囲を取り囲む**: 中央を主役にして、補助情報は周辺

#### ライティング演出
- **スポットライト感**: 中央に光が集まる演出
- **背景**: シルエット背後にうっすらと**スタジアム or 暗背景**
- **selected の演出**: focusEntry で切り替わるとき**フェード+スライド**

#### 比較対象の追加 (新)
primaryStat に「リーグ平均」「セ平均」「同年齢の OB」等の**比較値を併記**:
- 例: 「**WAR -0.4** (セ平均: 0.0)」
- これで**衝撃**が増す (「平均0なのに -0.4...」)

#### データスキーマの強化
```
旧: { players: [{id, name, number, label, silhouette, primaryStat, stats, comment}] }
新: 上記に primaryStat.compareValue を追加
    primaryStat: {
      label, value, isNegative,
      compareValue?: { value: "0.0", label: "セ平均" }
    }
```

**典型的な使い場面**:
- ranking で5人並べた後、**1人ずつ深掘り**
- 1選手の覚醒・不振動画の中盤
- 「なぜこの選手か」を主役として描くシーン

**避けるべき場面**:
- 比較が主目的 (versus_card や ranking が適切)
- データが浅い選手 (primaryStat と stats が埋まらないと薄く見える)

### 4.7 pitch_arsenal (球種パレット) — 役割: 「投手の手の内」を見せる

**ストーリーテリング上の役割**: 投手の**球種構成**を可視化

**メッセージ**: 「この投手の決め球は」「構成が変わった」「対右と対左で違う」

**アーク上の位置**: **中盤** (投手動画の深掘り)

**個性の方向**: **手の内感・武器カタログ感**。

**改修方針**:

#### 構成変化の比較 (新、強化)
既存の `comparePitches` (昨季 vs 今季) を**強化**:
- 球種ごとの**使用率の変化**を矢印で表示 (「直球 50% → 35% ▼15%」)
- 増えた球種は**緑の矢印 ↑**、減った球種は**赤の矢印 ▼**
- 視聴者が「**何が変わったか**」を瞬時に把握できる

#### 左右別比較 (新)
**右打者 vs 左打者**で配球を変えているか比較:

| 球種 | 対右打者 | 対左打者 | 違い |
|---|---|---|---|
| 直球 | 45% | 38% | -7% |
| カーブ | 15% | 22% | +7% |
| フォーク | 25% | 18% | -7% |

#### モード分岐
```
mode: "single"     // 単純な球種パレット (現状)
mode: "compare"    // 昨季 vs 今季 (既存)
mode: "vs_batter"  // 対右打者 vs 対左打者 (新)
```

#### データスキーマの拡張
```
{
  mode: "single"|"compare"|"vs_batter",
  pitches: [...],         // single 用
  comparePitches?: [...], // compare 用 (既存)
  vsBatter?: {            // vs_batter 用 (新)
    vsRight: [{type, ratio, avg}, ...],
    vsLeft: [{type, ratio, avg}, ...]
  }
}
```

**典型的な使い場面**:
- 投手の覚醒動画 (「フォークが化けた」、mode: compare)
- 投手の不振動画 (「ストレートが走らない」、mode: compare)
- 配球分析動画 (「対左に弱い理由」、mode: vs_batter)
- 投手対決動画 (球種構成の違い)

**避けるべき場面**:
- 打者動画 (使えない)

### 4.8 batter_heatmap (打者の打率ヒート) — 役割: 「打者の癖」を見せる

> 旧 `pitch_heatmap` をリネーム。投手向けではなく**打者向け**のヒートマップとする。

**ストーリーテリング上の役割**: 打者の**得意/不得意ゾーン**を可視化

**メッセージ**: 「このコースに強い」「アウトコースに弱い」「対右と対左で違う」

**アーク上の位置**: **中盤** (打者の癖・攻略法を見せる)

**個性の方向**: **戦術解説感**。投手目線で「ここに投げれば抑えられる」を直感的に。

**改修方針**:

#### 9エリア (3x3) の打率ヒート
従来の配球ヒートと同じビジュアル構造だが、**意味を変える**:
- 表示する数値は「**打者の打率**」(配球率ではない)
- 高打率ゾーン = 暖色 (赤・オレンジ)
- 低打率ゾーン = 寒色 (青・グレー)
- 平均打率付近 = 中性 (黄)

#### 左右別比較 (新)
打者は対右投手 / 対左投手で打率傾向が違う:

```
┌─ 対右投手 ─┐ ┌─ 対左投手 ─┐
│ .220 .280 .310 │ │ .180 .240 .290 │
│ .250 .310 .340 │ │ .210 .270 .320 │
│ .180 .220 .260 │ │ .160 .200 .240 │
└────────────┘ └────────────┘
```

#### モード分岐
```
mode: "single"    // 9エリアの打率 (右投・左投合計)
mode: "vs_handedness"  // 対右投 / 対左投 の左右並び (新)
```

#### データスキーマ
```
{
  mode: "single"|"vs_handedness",
  // single 用
  zones?: [9つの数字、左上から右下へ],
  // vs_handedness 用
  vsRight?: [9つの数字],
  vsLeft?: [9つの数字]
}
```

#### 既存 pitch_heatmap との関係
- レイアウト名を `batter_heatmap` に変更 (リネーム)
- 旧データ (投手の配球) は今後使わない
- 必要なら将来 pitch_heatmap を別名で復活

**典型的な使い場面**:
- 打者攻略動画 (「○○を抑えるならここに投げろ」)
- 打者の癖分析 (「対左に弱い」)
- 打席分析 (「この打席はゾーンを攻めた」)

**避けるべき場面**:
- 投手動画
- データが取れない場合 (NPB 公式に細分化された数字あるか要確認)

## 5. 動画テーマ → レイアウト選択の判断軸

### 5.1 主役レイアウトの選び方

| 動画テーマ | 主役レイアウト | 理由 |
|---|---|---|
| 朗報・覚醒 (1選手) | **timeline** | 変化を見せるのが核心 |
| 悲報・不振 (1選手) | **timeline** | 同上、悪化の経過 |
| 1対1対決 | **versus_card** | 対決そのもの |
| 順位/比較 (3人以上) | **ranking** | 序列を見せる |
| 個人深掘り | **player_spotlight** | 主役を見せる |
| チーム不振/好調 | **team_context (mode: single)** | チーム3要素ブロック |
| **巨人 vs セ平均** | **team_context (mode: compare)** | リーグ内ポジション |
| **巨人 vs 阪神** | **versus_card** | チーム1対1対決 |
| **セリーグ順位** | **ranking** (チームエントリ) | 順位表 |
| 編成論 | team_context → ranking | 全体→個別の順 |
| 投手攻略法 | pitch_arsenal | 球種構成の話 |
| 打者攻略法 | batter_heatmap | 打率ゾーンの話 |

### 5.2 動画内の遷移パターン (2-3 切替の王道)

| 主役 | 補助1 (中盤) | 補助2 (終盤) |
|---|---|---|
| ranking | player_spotlight (深掘り) | versus_card (結論) |
| timeline | radar_compare (全体像) | player_spotlight (個人) |
| player_spotlight | timeline (変化) | versus_card (対比) |
| versus_card | radar_compare (前提) | versus_card (再登場、結論) |
| team_context | ranking (個別順位) | player_spotlight (注目選手) |
| pitch_arsenal | radar_compare | versus_card |
| batter_heatmap | timeline (推移) | versus_card |

### 5.3 切替の禁則
- **5秒以内の連続切替** (視聴者が混乱)
- **データがないのに切替えるだけ**の演出
- **同じレイアウトに何度も戻る** (1回再登場までは OK)
- **全く異なるテーマのレイアウトを連発**
