# 構成プレイブック『数字で見るG党』

> scripts の構成・連続ルール・アウトロの正典。
> JSON生成時は毎回参照すること。

---

## 1. scripts 全体構造

### 1.1 必須ルール

| 項目 | ルール |
|---|---|
| **総 scripts 数** | **28〜30個** (★旧 35-45 から短縮、テンポ最優先★) |
| **id 連番** | id 1から連番、欠番禁止 |
| **動画長** | 28-30 id × 平均1.8秒 = **50-55秒** (60秒制限内) |
| **同 speaker 連続** | **最大2回まで** (★旧 4回から強化★) |
| **レイアウト切替** | 動画内 2-3 回 (15秒間隔以上、意味ある時のみ) |

### 1.2 4フェーズ構成

```
フェーズA (フック)     id 1            isCatchy: true
フェーズB (導入)       id 2-5          視聴者代弁・テーマ提示
フェーズC (本論)       id 6-25         現象+データの深掘り、ハイライト
フェーズD (アウトロ)   id 末尾2-3      自然な締め + 二択誘引
```

#### フェーズA (id:1) - フック=動画タイトル
- ★詳細は **hook-design.md** を必ず参照★
- isCatchy: true (固定 46px の大きな文字)
- 強調記号最低2箇所
- 主語/固有名詞/インサイト/3-4行×6-8字 必須

#### フェーズB (id:2-5) - 導入
- B (もえかちゃん) が視聴者代弁で振る
- A (数原さん) が「今日のテーマ」を提示
- ★A↔B の呼び合いがここで最低1回入る★
  - 例: B「数原さん!今日の○○ですね」/ A「そうですもえかちゃん」

#### フェーズC (id:6-25) - 本論
- 現象→数字→解釈の3段階で1指標を語る
- レイアウト切替で飽きを防ぐ (2-3回)
- ハイライト機能で「今この数字を話してます」を視覚化

#### フェーズD (id:末尾2-3) - アウトロ
- 自然な締め or 文脈ある二択
- ★末尾2 scripts は「両論併記+選択を視聴者に投げる」のが理想★
- 詳細は本ファイル「3. アウトロ設計」参照

## 2. ★同 speaker 連続ルール (最重要、視聴維持率に直結)

### 2.1 ルール

- ★同キャラ連続は **最大2回**まで★ (★旧ルール「4回まで」は廃止★)
- 3回以上は禁止 (一人の独演会で視聴者が離脱)
- A連続2回 → B → A連続2回 → B... の **リズム** を基本とする
- 1id が長くなりそうなら **短く分割して2idにする方が良い** (視聴維持率が上がる)
- ただし「内容が薄いまま2連続」も避ける。**情報密度を保つ**こと

### 2.2 良い例: A説明 → B驚き → A追い打ち のリズム

```
A1: "純粋な長打力を示す「IsoP」を見てください"
A2: "昨季は【.095】、今季は【.172】に大幅上昇"
B:  驚きの相槌「え?ほぼ倍ですね!」
A3: "リーグ平均超えで、これが主軸の証です"
```

### 2.3 悪い例: A4連続 (独演会)

```
A1: "IsoPの話をします"
A2: "IsoPとは長打率から打率を引いた指標で..."
A3: "今季の岡本は.172です"
A4: "去年の.095から大きく上がってます"
   ↑ Bが入らない、視聴者が「説教感」で離脱
```

### 2.4 B の相槌レパートリー

短くて良い、独立 1id にする:
- "え？" / "まじで？" / "なるほど" / "それは…"
- "ほぼ倍ですね!" / "驚きました"
- 共感: "わかります、それ感じてました"
- 質問: "それって、つまりどういうことですか?"
- 異論: "でも先週の試合、活躍してませんでした?"

## 3. アウトロ設計 (★末尾2-3 scripts、文脈ある二択)

### 3.1 自然な締めパターン (推奨)

末尾2-3 scripts は **「両論併記+選択を視聴者に投げる」**:

```
A (倒数2): "結論。○○で△△を崩し、××がねじ伏せるか"
A (倒数1): "それとも『天敵』○○の一発に沈むか"
B (末尾):  "皆さんは○○派?△△派?"
```

→ 「勝つ」「快投」と断定せず、**両方の可能性をデータで提示してから視聴者に問う**。
これがコメント誘発+視聴維持率を両立する最強パターン。

### 3.2 NG: 唐突な二択 (廃止ルール)

```
B: "皆さんは10本派?20本派?"  (前振りなし)
```
→ 文脈なしの二択は離脱誘発。必ず**直前で両論併記**してから問う。

### 3.3 NG: 定型句

```
B: "コメントで教えてください"  ← 定型句、離脱率最悪
B: "いいねと登録お願いします"   ← 同上
```

★ 「コメントで教えて」は実投稿動画で離脱率が顕著に高かった鉄板NGパターン。

### 3.4 二択疑問の良い例集

| 動画テーマ | 二択 |
|---|---|
| 投手の試合プレビュー | 「快投を信じる派？天敵に注意派？」 |
| 打者の覚醒 | 「本物だと思う派？まだ早い派？」 |
| 監督采配 | 「采配支持派？反対派？」 |
| 編成論 | 「2軍降格派？我慢使い続ける派？」 |
| 不振選手 | 「化ける派？このまま終わる派？」 |

## 4. レイアウト切替パターン

### 4.1 動画テーマ別パターン

| テーマ | 推奨レイアウト遷移 |
|---|---|
| 朗報/覚醒 | radar_compare → timeline → ranking |
| 悲報 | radar_compare → batter_heatmap → team_context |
| 対決 (投手 vs 投手等) | versus_card → radar_compare → player_spotlight |
| チーム/采配 | team_context → ranking → player_spotlight |
| 順位/比較 | ranking → player_spotlight → versus_card |
| 投手分析 | radar_compare → pitch_arsenal → player_spotlight |
| 打者分析 | radar_compare → batter_heatmap → timeline |

### 4.2 切替ルール

- 切替最低 **15秒間隔** (5 script以上同じレイアウト)
- 切替 script に **se: "transition_swoosh"** 追加
- レイアウト切替時は **layoutType フィールド** で指定

### 4.3 パターンの2本連続使用禁止

同一パターンの2本連続使用は禁止 (動画ごとに変える)。

## 5. ハイライト継続ルール

### 5.1 ハイライト = 「今この数字を話してます」を視覚化

text 内で指標について話している期間中、`highlight: "k_9"` のように指定する。
これが続いてる間、レイアウト内の該当行/数値が強調される。

### 5.2 継続ルール

```
id:9  highlight: "k_9"      "K/9を見てください"
id:10 highlight: "k_9"      "井上は驚異の【11.20】"
id:11 highlight: "k_9"      "1試合で11個も三振"
id:12 highlight: なし        "話題が逸れたので外す"
id:13 highlight: "bb_9"     "次はBB/9を見てください"
```

★ 同じ指標の話が複数 script に渡る場合、**全ての該当 script に同じ highlight** を設定する。

## 6. テキスト強調ルール

### 6.1 強調記号の意味

| 記号 | 色 | 用途 | 例 |
|---|---|---|---|
| **【】** | 黄色拡大 | 数字・試合名・選手名 | `【11.20】` `【巨人vsDeNA】` |
| **「」** | オレンジ拡大 | 指標名・キーワード | `「K/9」` `「鍵」` |
| **『』** | 赤拡大 | 衝撃ワード・敵 | `『天敵』` `『真逆』` |

### 6.2 適用ガイド

- **数値は必ず【】で囲む** (例: `【.285】` `【11.20】`)
- **指標名は「」で囲む** (例: `「OPS」` `「K/9」`)
- **衝撃・警告ワードは『』で囲む** (例: `『天敵』` `『懸念』`)

### 6.3 1 script あたりの強調目安

- 通常 script (id:2以降): **1〜2箇所**
- フック (id:1): **最低2箇所** (★詳細は hook-design.md★)

## 7. textSize ルール

各 script に textSize を指定 (フェーズB-D で使用):

| サイズ | 用途 |
|---|---|
| `xl` | 衝撃の一言 (例: B「えっ!?」、最大インパクト) |
| `l` | 重要な数字や決め台詞 |
| `m` | 標準 (デフォルト) |
| `s` | 補足・長めの説明 |

★ フェーズA (id:1, isCatchy:true) は固定 46px なので textSize 不要。

## 8. SE (効果音) の使い方

| SE 名 | 使う場面 |
|---|---|
| `hook_impact` | id:1 (フックの登場) |
| `stat_reveal` | 重要な数値を出す前 |
| `highlight_ping` | ハイライト切替時 |
| `shock_hit` | B の驚き ("えっ!?" 等) |
| `success_chime` | 結論 (朗報) |
| `warning_alert` | 警告・敵の登場 |
| `transition_swoosh` | レイアウト切替時 |
| `outro_fade` | 末尾 |

★ SE は1動画で **5〜8回程度**が適量。多用すると煩い。

## 9. 自己チェック (出力前)

JSON生成後、以下を全部 YES にすること:

1. ★ scripts 総数が **28-30個** に収まっているか?
2. ★ 同 speaker の連続発言が **最大2回まで** か?
3. ★ A↔B の呼び合いが冒頭付近 (id:2-5) で最低1回入っているか?
4. レイアウト切替が 2-3 回あるか?
5. アウトロが「自然な締め」or「前振りある二択誘引」になっているか?
6. NG ワード (本ファイル + audience-and-language.md) が含まれていないか?
7. ハイライト継続ルールを守っているか? (同指標の話してる間は highlight 継続)
8. 数値は【】、指標名は「」、衝撃ワードは『』で囲っているか?

★ この8つは **structure-playbook.md** からの厳しい自己チェック。すべて YES で出力可能。
