# 構成プレイブック『数字で見るG党』

> scripts の構成・連続ルール・アウトロの正典。
> JSON 生成時は毎回参照すること。

---

## 1. scripts 全体構造

### 1.1 必須ルール

| 項目 | ルール |
|---|---|
| 総 scripts 数(縦長 9:16) | 60 秒以内に収まる範囲で柔軟(目安 20-30、固定ではない) |
| 総 scripts 数(横長 16:9) | 動画長に応じて柔軟(2-3 分目安 30-50) |
| id 連番 | id 1 から連番、欠番禁止 |
| 同 speaker 連続 | 最大 4 回まで(5 回以上 NG) |
| 同 scenePreset 連続 | 最大 4 ID まで(5 ID 以上 NG) |
| 同 se 連続 | 最大 4 ID まで(5 ID 以上 NG) |
| レイアウト切替 | 動画内 2-3 回(15 秒間隔以上、意味ある時のみ) |

### 1.2 4 フェーズ構成

```
フェーズA (フック)     id 1                   isCatchy: true
フェーズB (導入)       id 2 から数 ID         視聴者代弁・テーマ提示・呼び合い
フェーズC (本論)       中盤                   現象+データの深掘り、ハイライト
フェーズD (アウトロ)   末尾 2-3                自然な締め + 二択誘引
```

#### フェーズA (id:1) — フック=動画タイトル

★id:1 は単なる「最初のテロップ」ではなく **動画タイトル**★。ショート動画は最初の 0.5 秒で「これは何の話か」を伝えないとスワイプされる。
ここで失敗すると、後がどんなに良くても継続率は 20% を切る。

##### 必須要件
- `isCatchy: true`、`textSize: "xl"`、`se: "hook_impact"`
- 強調記号(【】「」『』)最低 2 箇所
- 主語/固有名詞/インサイト/3-4 行 × 6-8 字
- ★id:1 は分割しない★(タイトル独立、speech は短いフック 1-2 秒)

##### ターゲット視聴者の二層構造(★最重要★)
メインターゲットはコアファンだが、**id:1 だけはミーハー層もスワイプしない設計**にする:
- **本論(id:2 以降)** はコアファン基準で深く語る(現象+データ)
- **id:1(動画タイトル)** はミーハーが見ても「自分の話だ」と分かる

★id:1 で OPS/WAR を出すのは絶対 NG★(ミーハー脱落)。

##### インサイト vs 願望(★絶対ルール★)
「数字で見るG党」は **データ分析チャンネル** であって応援チャンネルではない。願望ベースの言葉は即スワイプされる。

- **インサイト**: データから導かれる「観戦する上での鍵・条件・分かれ目」
  - 例: 「佐野の前に走者を出すかどうかが分かれ目」
  - 例: 「岡本の打率.228は本物ではない、IsoP.172が示す通り主軸」
- **願望**: データに基づかない「こうなってほしい」という応援
  - 例: 「今日も巨人が勝つ」「井上が三振でねじ伏せる」「期待したい」「快投を信じる」

##### 願望ワード(★id:1で使ったらボツ、本文でも極力避ける★)

| カテゴリ | 願望ワード |
|---|---|
| 結論型 | 「勝つ」「勝てる」「勝てます」 |
| 賞賛型 | 「快投」「圧勝」「ねじ伏せる」(根拠なしの場合) |
| 期待型 | 「期待」「信じる」「頑張れ」「願う」「楽しみ」 |
| 推測型 | 「○○してほしい」「○○ありそう」「○○できそう」 |

★ ただし末尾アウトロの「○○か、それとも△△か」のような **両論併記** は願望ではない(二択誘引として OK)。

##### 願望 → インサイト 変換例

| 願望 (NG) | インサイト (OK) |
|---|---|
| 「今日も勝つ」 | 「鍵は被安打22の石田を連打で崩すこと」 |
| 「井上が三振でねじ伏せる」 | 「井上のK/9 11.20が活きるのは佐野前に走者出さない条件下」 |
| 「快投に期待」 | 「直近2試合連続失点中、今日が立て直しの試金石」 |
| 「岡本に頼む」 | 「岡本の打率.228は ”本物の不振” か ”主軸の証” か」 |

##### 失敗パターン集(★実例★)

| ダメな id:1 | 何が悪いか | どう直すか |
|---|---|---|
| 「奪三振【11.2】\n四球【ゼロ】」 | 主語不明・誰の話か不明 | 「【井上温大】\n奪三振【11.2】」 |
| 「三振【11.2】\n今日勝てる?」 | 願望、固有名詞無し | 「【巨人vsDeNA】井上の鍵は『佐野』」 |
| 「井上温大\n奪三振【11.2】」 | データだけでインサイト無し | 「【井上】三振【11.2】\n鍵は『天敵』佐野」 |
| 「井上温大の三振【11.2】\n佐野さえ抑えれば勝てる」 | 願望「勝てる」が混入 | 「井上温大\n『天敵』佐野の前に\n走者出すな」 |
| 「本当の現在地」 | 抽象煽り、何が言いたいか不明 | 具体的な選手名 + 具体的問題に書き換え |

##### ギャップ作り(★即スワイプ防止★)

★鉄則: id:1 で全結論を言わない★

❌ NG: 「岡本が3割打者になった理由は配球変化でした」 → 結論が見えて離脱
✅ OK: 「岡本和真【.305】覚醒の理由」 → 「なぜ?」と思わせて続きを見たくなる
✅ OK: 「戸郷翔征 奪三振封印!?」 → 通常イメージと逆 → 「なぜ?」

##### 序盤の最速回収(期待値コントロール)

id:1 で提示した謎の核心データは **id:4-8 までに最速で提示**:

```
id:1   「奪三振封印!?」
id:2-3 状況設定
id:4   「奪三振率は前年比 -3.2」 ← ここで核心提示(期待回収)
id:5-N 多角的に深掘り
```

❌ NG: id:1 で「奪三振封印!?」と言いながら、id:8 まで打率の話 → 期待ズレで離脱

##### id:1 の speech 短縮テクニック

text は完全形(3-4 行 × 6-8 字)だが、**speech はテンポ優先で短く切ってよい**(★id:1 のみ例外★)。

理由:
- 動画開始 2 秒以内が「スクロール継続/離脱」の分岐点
- text 全文読み上げると 4-5 秒かかり、その間に離脱されるリスク
- **speech は印象的なフレーズだけ**(1-2 秒で言える 8-15 文字程度)で十分

例:
```
text 全文: 「岡本和真\n【.305】\n覚醒の理由」

❌ NG (長すぎ): "speech": "おかもとかずまの今季の打率はさんわりごぶ、覚醒の理由を解説します。"
✅ OK (短く印象的): "speech": "岡本和真。【.305】の覚醒。"
```

判断基準:
- speech は **2 秒以内に終わる長さ**(約 12-15 文字)
- 主語(固有名詞)+ 核心ワードだけ
- 説明調の動詞は省く
- 「。」で間を作って印象づけ

##### playerType / silhouetteType 整合性

| テーマ | playerType | silhouetteType | hook 例 |
|---|---|---|---|
| 投手分析 | pitcher | pitcher_left/right/set/catcher | 「【井上温大】\n奪三振【11.2】」 |
| 打者分析 | batter | batter_left/right/stance/runner | 「【岡本和真】\n打率.228でも『主軸』」 |
| チーム/監督/采配 | team | team_stadium / team_huddle | 「巨人の犠打\n【80→18】激減」 |

★監督ネタで打者シルエットだと「誰?」で離脱★。テーマと型を一致させる。

##### hook と stats(画面下 4 項目)の意味的一致
- hook が「チームのバント激減=大正解」 → stats=犠打数/得点/勝率/勝ち越し
- hook が「個人の IsoP 急上昇」 → stats=AVG/OPS/HR/RBI

★hook と stats が矛盾するとミーハー脱落★(例: hook「大正解」なのに stats AVG.228 → 何の話?)


#### フェーズB (id:2-5) — 導入
- B(もえかちゃん)が視聴者代弁で振る
- A(数原さん)が「今日のテーマ」を提示
- ★A↔B の呼び合いがここで両方向必須★
  - 例: B「数原さん!今日の○○ですね」/ A「そうですもえかちゃん」

#### フェーズC — 本論
- 現象 → 数字 → 解釈の 3 段階で 1 指標を語る
- レイアウト切替で飽きを防ぐ(2-3 回)
- ハイライト機能で「今この数字を話してます」を視覚化
- 相槌(「なるほど」「えっ」「なんと」)は独立 id にして OK(テンポが出る)

#### フェーズD (末尾 2-3) — アウトロ
- 自然な締め or 文脈ある二択
- ★末尾 2 scripts は「両論併記+選択を視聴者に投げる」のが理想★
- 詳細は本ファイル「3. アウトロ設計」参照

## 2. text と speech のルール

- ★id:2 以降は text と speech の文章内容を完全一致させる★(表記のみ違い OK: text `75%` ⇄ speech `ななじゅうごぱーせんと`)
- id:1 は例外で speech は短いフック 1-2 秒(text 全文を読まない、離脱防止)
- 改行 `\n` は text のみ / 句点「。」は text 省略
- ★speech 内の読点「、」は極力削る★ → 間が必要な箇所は全角スペース「　」で代用
- 1 ID あたり: 縦長 9:16 → text 3-12 字 × 3-4 行(合計 12-30 字)/ 横長 16:9 → text 8-20 字 × 1-2 行
- 超える場合は ID 分割

## 3. アウトロ設計(末尾 2-3 scripts、文脈ある二択)

### 3.1 自然な締めパターン(推奨)

末尾 2-3 scripts は「両論併記+選択を視聴者に投げる」:

```
A (倒数2): "結論。○○で△△を崩し、××がねじ伏せるか"
A (倒数1): "それとも『天敵』○○の一発に沈むか"
B (末尾):  "皆さんは○○派?△△派?"
```

→ 「勝つ」「快投」と断定せず、両方の可能性をデータで提示してから視聴者に問う。
これがコメント誘発+視聴維持率を両立する最強パターン。

### 3.2 NG: 唐突な二択

```
B: "皆さんは10本派?20本派?"  (前振りなし)
```
→ 文脈なしの二択は離脱誘発。必ず直前で両論併記してから問う。

### 3.3 NG: 定型句

```
B: "コメントで教えてください"  ← 定型句、離脱率最悪
B: "いいねと登録お願いします"  ← 同上
```

「コメントで教えて」は離脱率が顕著に高い鉄板 NG パターン。

### 3.4 二択疑問の良い例集

| 動画テーマ | 二択 |
|---|---|
| 投手の試合プレビュー | 「快投を信じる派?天敵に注意派?」 |
| 打者の覚醒 | 「本物だと思う派?まだ早い派?」 |
| 監督采配 | 「采配支持派?反対派?」 |
| 編成論 | 「2軍降格派?我慢使い続ける派?」 |
| 不振選手 | 「化ける派?このまま終わる派?」 |

## 4. レイアウト切替パターン

### 4.1 動画テーマ別パターン

| テーマ | 推奨レイアウト遷移 |
|---|---|
| 朗報/覚醒 | radar_compare → timeline → ranking |
| 悲報 | radar_compare → batter_heatmap → team_context |
| 対決(投手 vs 投手等) | versus_card → radar_compare → player_spotlight |
| チーム/采配 | team_context → ranking → player_spotlight |
| 順位/比較 | ranking → player_spotlight → versus_card |
| 投手分析 | radar_compare → pitch_arsenal → player_spotlight |
| 打者分析 | radar_compare → batter_heatmap → timeline |

### 4.2 切替ルール

- 切替最低 15 秒間隔(5 script 以上同じレイアウト)
- 切替 script に `se: "transition_swoosh"` 追加
- レイアウト切替時は `layoutType` フィールドで指定

### 4.3 パターンの 2 本連続使用禁止

同一パターンの 2 本連続使用は禁止(動画ごとに変える)。

## 5. ハイライト継続ルール

### 5.1 ハイライト = 「今この数字を話してます」を視覚化

text 内で指標について話している期間中、`highlight: "k_9"` のように指定する。
これが続いている間、レイアウト内の該当行/数値が強調される。

### 5.2 継続ルール

```
id:9  highlight: "k_9"      "K/9を見てください"
id:10 highlight: "k_9"      "井上は驚異の【11.20】"
id:11 highlight: "k_9"      "1試合で11個も三振"
id:12 highlight: なし        "話題が逸れたので外す"
id:13 highlight: "bb_9"     "次はBB/9を見てください"
```

同じ指標の話が複数 script に渡る場合、全ての該当 script に同じ highlight を設定する。

### 5.3 versus_card / player_spotlight の行強調(基本成績の迷子防止)

`highlight` は HighlightCard だけでなく、通常表示でも行強調が発火する:

- `comparison.label` と `versus_card.categoryScores[].label` が一致 → その行が脈動 + 数字拡大
- `comparison.label` と `player_spotlight.primaryStat.label` or `stats[].label` が一致 → 該当カードが脈動

これにより、A が「井上は驚異の【11.20】」と話してる時、画面のどこに 11.20 があるかが一目瞭然になる。

- `highlight` をしっかり指定すること。指定が無いと行強調は発火しない。
- ラベル一致は完全一致 or 双方向部分一致(case-insensitive)。
  - 例: `comparison.label="K/9"` / `categoryScores[].label="奪三振率(K/9)"` でも一致。

## 6. smartLoop(シームレスループ前提)

トップレベルの `smartLoop: true`(デフォルト)で、末尾 → 冒頭にシームレスループ。

このため:
- アウトロの「今日のまとめ」「チャンネル登録お願い」静止画は表示されない
- 末尾 2-3 scripts は会話の自然な締めでちょうど終わるよう設計
- 視聴者は「あれ、もう 2 週目に入ってた」と気づかないループを目指す
- `outroCta` は出力に含めても表示されない(互換性のため受け取るが無視)
- 末尾の `se: "outro_fade"` も effectively には鳴らない

逆に **smartLoop を OFF にしたい場合**(詳細結果見せたい等)は明示的に `smartLoop: false` を指定。
基本は省略 or `true` で OK = ループ前提で締めを設計する。

## 7. ミステリー型構成(アハ体験の創出)

### 推奨展開順序(チャンネル憲章)

```
① 現象(序盤)    : 現場で起きている事象を提示
   例: 「アウトの取り方が変わった」
       「有利カウントで打たれる」
       「球速は上がってないのに被打率激減」

② 衝撃データ    : 現象の裏付けとなる数字を叩きつける
   例: 「P/IP が前年比 -1.2 球」
       「初球ストライク率 62% → 75%」
       「対左 IsoP が .045 → .172」

③ 権威の言葉    : OB/解説者/本人コメントで補強(一次ソース)
   例: 「○○氏は『出力を抑えた配球が効いてる』と分析」
       「本人は『フォームより配球の意図を変えた』と語る」

④ 根本原因      : 終盤で「なぜそうなったか」の真因を明かす
   例: 「実は極度の打撃不振による焦りからの反動」
       「データ班の助言で配球バリエーションを増やした」
```

### NG パターン

❌ id:1 で全結論 → 視聴者「もう答え見たから離脱」
❌ id:1 で「打率の話」と言って id:8 まで配球の話 → 期待ズレで離脱
❌ id:2 から根本原因を語る → アハ体験ゼロ、教科書朗読

### OK パターン

✅ id:1: 「奪三振を封印!?」(ギャップ・謎)
✅ id:2-3: 状況設定(キャラ呼び合い)
✅ id:4-7: 核心データを早めに提示(期待値の回収)
✅ id 中盤: 多角的に深掘り(現象/データ/権威の言葉)
✅ id 終盤: 根本原因の解明(アハ体験)
✅ id 末尾: 締め

## 8. テロップの文字量とテンポ

### 1 ID あたりの文字量

- 縦長 9:16 → text 3-12 字 × 3-4 行(合計 12-30 字)
- 横長 16:9 → text 8-20 字 × 1-2 行
- 上限を超える場合は ID 分割推奨 → スピーディーな展開
- 1 秒以下のシーンが続くと脳が処理しきれないので、文字量と発話時間を整合させる

### 句点禁止ルール(★絶対★)

- text には句点「。」を使わない
  - テロップ視認は瞬間的で、句点は冗長 + 行末違和感
  - 改行(`\n`)で意味の区切りを表現する
- speech(TTS)には句点 OK(自然な間として推奨)

### 例

❌ NG:
```json
{ "text": "今季の打率は3割。\n突出した数字です。", "speech": "今季の打率は3割。突出した数字です。" }
```

✅ OK:
```json
{ "text": "今季の打率は\n【3割】\n突出した数字", "speech": "今季の打率は3割　突出した数字です" }
```

## 9. YouTube 投稿情報生成(オプション、動画依頼時のみ)

★ ユーザーから「動画一式」「YouTube に投稿する情報も」と依頼された場合のみ、JSON 出力と併せてこのセクションの形式で YouTube 投稿情報を出力する。
★ JSON のみ依頼の場合は出力不要。
★ ショート(9:16)と横長(16:9)で SEO 戦略が違う。

### 9.1 動画タイトル(YouTube SEO 用、id:1 とは別)

#### 役割
YouTube の検索・関連動画で見つけられるためのタイトル。id:1(動画内テロップ)とは別物。

#### 文字数
- **30-60 字**(40 字前後が最適)
- 検索結果での表示は 50 字程度で切れる、重要キーワードは前半に

#### 構造テンプレ
```
【<選手名 or チーム名>】<具体数値>+<物語キーワード>|<補助フレーズ>
```

#### 縦長(ショート)向け例
- `【巨人】戸郷翔征、防御率7.20の真相|消されたフォークの謎 #Shorts`
- `【数字で見る】岡本和真、本塁打率1/12.5の異次元 #巨人 #Shorts`

#### 横長向け例
- `【巨人】データで読み解く戸郷翔征の現在地|質の高い直球とフォークの課題`
- `【深掘り解説】岡本和真がメジャー速球を打ち砕いた『14cm』の秘密`

#### NG パターン
- 抽象タイトル(「驚愕の真実」「衝撃のデータ」単体)
- 主語不明(「○○の正体」だけ)
- 全角記号過多(`!!?!?!` 等)

### 9.2 概要欄

#### 役割

### 9.3 ハッシュタグ

#### 役割
YouTube 内検索・トレンド誘導用。概要欄の最後に並べる。

#### 個数
- **5-15 個**(多すぎ NG、関連性低いタグは逆効果)

#### 必須タグ(全動画共通)
- `#数字で見るG党`(チャンネル独自)
- `#巨人` / `#読売ジャイアンツ`
- `#プロ野球` / `#NPB`
- `#データ分析` / `#セイバーメトリクス`

#### 動画別タグ
- `#<選手名>`(漢字フル + ひらがな両方推奨: `#戸郷翔征` `#とごうしょうせい`)
- `#<対戦相手>` (相手チーム動画なら)
- `#Shorts`(縦長動画のみ)

#### 例(戸郷 縦長)
`#数字で見るG党 #巨人 #戸郷翔征 #プロ野球 #データ分析 #セイバーメトリクス #Stuff #ピッチング #Shorts`

### 9.4 タグ(SEO 用、視聴者非表示)

#### 役割
YouTube アルゴリズムへのヒント、関連動画誘導。

#### 個数
- **15-30 個**(YouTube 推奨)

#### 構成
- ハッシュタグの内容を全て含める
- 加えて関連検索ワード:
  - `<選手名> 成績`
  - `<選手名> データ`
  - `<選手名> 不調 / 復活 / 覚醒`
  - `巨人 投手陣` / `巨人 打線` 等
  - `セイバーメトリクス 解説`
  - チャンネル名のバリエーション

#### 例
```
数字で見るG党, 巨人, 読売ジャイアンツ, プロ野球, NPB, データ分析,
セイバーメトリクス, Stuff+, Location+, ピッチング, 戸郷翔征, 
戸郷翔征 成績, 戸郷翔征 不調, 戸郷翔征 データ, 巨人 投手陣,
2026年 巨人, 阿部慎之助, ローテーション, フォーク, 直球, 制球
```

### 9.5 サムネ案(★YouTube 縦動画/横動画ともに重要)

#### 役割
クリック率(CTR)に最大の影響。文字 + 画像 + 表情 + 数字。

#### 構造
1. **大きい数字**(画面の 30-40%)
2. **強調テキスト**(画面の 20-30%)
3. **選手画像 or シルエット**
4. **背景演出**(派手・静物・ニュース風 等)

#### 案 1: 数字 + 矛盾型
```
【107.3】← 巨大表示
「球は来てる」 ← 上部
「なのに、なぜ?」 ← 下部
+ 選手画像(投球モーション、悔しい表情)
+ 背景: 黒+赤グラデーション
```

#### 案 2: ギャップ提示型
```
左: 2024年「絶対王者」 1.95
右: 2026年「???」 7.20
+ 選手 2 枚並べ
+ 背景: 上下分割(明 vs 暗)
```

#### 案 3: ファン心理代弁型
```
「消されたフォーク」 ← 大きく
「巨人ファンが知るべき真実」 ← 小さく
+ 選手画像 + フォーク軌道のイラスト
+ 背景: 紺+白
```

#### サムネ NG
- 文字が小さい(スマホで読めない、20 字以内)
- 顔写真なし(視認性下がる)
- 過度な煽り(「これは絶対見て!」等は逆効果)

### 9.6 固定コメント

#### 役割
動画投稿後、自分のコメント欄に固定で表示。視聴者の議論を誘導し、エンゲージメント向上。

#### 文字数
- **100-300 字**

#### テンプレート

```

今回の【<題材>】、いかがでしたか?

▼皆さんに聞きたい
・<質問 1: 動画内の論点を投げる>

コメント欄で皆さんの意見を聞かせてください👇

▼関連動画もぜひ
<関連動画 URL>

#<主要ハッシュタグ 3 個>
```
