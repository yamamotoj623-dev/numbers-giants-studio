# Grok ワークスペース指示 (プロジェクト共通文脈)

## このワークスペースは何か

YouTube Shorts チャンネル『**数字で見るG党**』の動画制作プロジェクト。
プロ野球データ分析・読売ジャイアンツ専門・縦動画 (9:16) 60秒未満。

中で 4 つのカスタムエージェントを使い分け:
- `数字で見るG党マスター` (全自動)
- `数字で見るG党 リサーチ` (詳細データ調査)
- `数字で見るG党 監修者` (JSON レビュー)
- `数字で見るG党 トレンド` (旬ネタ発掘)

各エージェントの詳細指示は別途設定済み。これは**全エージェント共通の前提**。

## ターゲット視聴者

- 30-60代男性 巨人ファン、ミーハー層 6 : コア層 4
- スマホで TikTok 的フリック視聴
- ★ミーハー層が脱落しない設計が最優先★

## 実視聴データの法則 (94,417再生 / 33本)

| 要因 | 良い | 悪い |
|---|---|---|
| hook | 13字以下 (継続23.4%) | 17字以上 (継続14.1%) |
| 動画長 | 60秒未満 (維持34.7%) | 70秒超 (維持26.6%) |
| 強ワード位置 | 1〜2行目に数字+対比 | 3行目に結論 (視聴16%実例) |
| アウトロ | 二択疑問でブツ切り | 「コメントで教えて」(定型句) |
| シルエット整合 | 監督→team_stadium | 監督→打者 |

## 共通の禁止事項

- NGワード: 「本当の」「現在地」「可能性」「だった!?」「驚愕の」「ヤバい」「コメントで教えて」
- ファン感情論で曖昧化 (「期待したい」「応援したい」等)
- 訓練データ記憶ベースの数字 (必ず web_search/x_search で最新)
- 完璧主義による発信遅延

## 添付ファイル (全エージェント参照)

   speech フィールド生成時必ず参照。例: 泉口→いずぐち、左腕→さわん
2. **gemini-custom-prompt.md** - Gemini側JSON生成プロンプト (schemaVersion 5.0.0)
3. **workflow.md** - 動画制作フロー (方式A/B)

## アプリ側仕様 (前提知識)

- React + Vite アプリ『数字で見るG党 Studio v5.5.0』
- JSON 入力 → プレビュー → mp4 ダウンロード
- 9レイアウト: radar_compare / timeline / luck_dashboard / spray_chart / pitch_heatmap / versus_card / pitch_arsenal / team_context / ranking
- 動画内 2-3 レイアウト切替可 (script.layoutType)
- TTS: Gemini 3.1 Flash (Charon=A, Puck=B)
- アプリ側でも yomigana 自動置換あるが、AI生成時点で正確に書く方が望ましい

## エージェント運用方針

### 普段 (方式A)
マスターに依頼 → モード自動判定 → 完成JSON。
例「増田陸の四球ゼロで動画作って」「最近の旬ネタは?」

### 難案件 (方式B)
炎上リスク・繊細な擁護で 4枠使い分け:
トレンド → リサーチ → (Gemini で生成) → 監修者レビュー

## 共通トーン

- A (アナリスト): 「私」「〜です」断定、予防線張らない
- B (視聴者代弁): 「〜じゃないですか」、素人質問でA に翻訳させる
- 数字で斬る、感情論で締めない、二択疑問で終わる
