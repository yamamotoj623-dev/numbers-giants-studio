## 1. id:1 の定義: 動画タイトル

### 1.1 id:1 とは何か

**id:1 は単なる「最初のテロップ」ではなく、動画タイトル**。

ショート動画は最初の **0.5秒** で「これは何の話か」を視聴者に伝えないとスワイプされる。
ここで失敗すると、後がどんなに良くても **継続率は20%を切る**。

### 1.2 id:1 の役割

| 役割 | 説明 |
|---|---|
| **動画タイトル** | サムネイル代わり、検索でも表示される |
| **スワイプ防止フック** | 0.3秒で「自分の話だ」と思わせる (自分ごと化) |
| **視聴の方向づけ** | 何を語る動画かを宣言する |

### 1.3 ★最重要: ターゲット視聴者の二層構造

メインターゲットはコアファンだが、**id:1 だけはミーハー層もスワイプしない設計**にする:

- **本論 (id:2 以降)** はコアファン基準で深く語る (現象+データ)
- **id:1 (動画タイトル)** はミーハーが見ても「自分の話だ」と分かる

**この2層を区別すること**。id:1 で OPS/WAR を出すのは絶対NG (ミーハー脱落)。

## 4. 失敗パターン集 (実際にユーザーが却下した例)

| ダメな id:1 | 何が悪いか | どう直すか |
|---|---|---|
| 「奪三振【11.2】\n四球【ゼロ】」 | 主語不明・誰の話か不明 | 「【井上温大】\n奪三振【11.2】」 |
| 「三振【11.2】\n今日勝てる？」 | 願望、固有名詞無し | 「【巨人vsDeNA】井上の鍵は『佐野』」 |
| 「井上温大\n奪三振【11.2】」 | データだけでインサイト無し | 「【井上】三振【11.2】\n鍵は『天敵』佐野」 |
| 「井上温大の三振【11.2】\n佐野さえ抑えれば勝てる」 | 願望「勝てる」が混入 | 「井上温大\n『天敵』佐野の前に\n走者出すな」 |
| 「無四球の石田は連打で攻略\n佐野警戒で今日も巨人が勝つ」 | 「巨人が勝つ」は願望 | 「鍵は『連打』と『佐野封じ』」 |
| 「本当の現在地」 | 抽象煽り、何が言いたいか不明 | (具体的な選手名 + 具体的問題に書き換える) |

## 6. インサイト vs 願望 (★最重要★)

「数字で見るG党」は**データ分析チャンネル**であって応援チャンネルではない。
願望ベースの言葉は視聴者に「で、結局なに?」と思われ、即スワイプされる。

### 6.1 定義

- **インサイト**: データから導かれる「観戦する上での鍵・条件・分かれ目」
  - 視聴者が試合を見る時に**注目すべき具体的な観点**を提供する
  - 例: 「佐野の前に走者を出すかどうかが分かれ目」
  - 例: 「無四球の石田は被安打22で崩せる」
  - 例: 「岡本の打率.228は本物ではない、IsoP.172が示す通り主軸」

- **願望**: データに基づかない「こうなってほしい」という応援
  - 視聴者の応援感情と重複するだけで**新しい情報を提供しない**
  - 例: 「今日も巨人が勝つ」「井上が三振でねじ伏せる」「期待したい」「快投を信じる」

### 6.2 願望ワード (id:1で使ったらボツ、本文でも極力避ける)

| カテゴリ | 願望ワード |
|---|---|
| 結論型 | 「勝つ」「勝てる」「勝てます」 |
| 賞賛型 | 「快投」「圧勝」「ねじ伏せる」(根拠なしの場合) |
| 期待型 | 「期待」「信じる」「頑張れ」「願う」「楽しみ」 |
| 推測型 | 「○○してほしい」「○○ありそう」「○○できそう」 |

★ ただし末尾アウトロの「○○か、それとも△△か」のような**両論併記**は願望ではない (二択誘引としてOK)。詳細は **structure-playbook.md** 参照。

### 6.3 願望 → インサイト 変換テンプレート

| 願望 (NG) | インサイト (OK) |
|---|---|
| 「今日も勝つ」 | 「鍵は被安打22の石田を連打で崩すこと」 |
| 「井上が三振でねじ伏せる」 | 「井上のK/9 11.20が活きるのは佐野前に走者出さない条件下」 |
| 「快投に期待」 | 「直近2試合連続失点中、今日が立て直しの試金石」 |
| 「岡本に頼む」 | 「岡本の打率.228は ”本物の不振” か、IsoP.172が示す通り ”主軸の証” か」 |
| 「打線爆発しそう」 | 「無四球の石田を崩す唯一の道は連打、ヒット22本以上のリズム」 |

## 7. id:1 の hook 構造 — stats との一致

### 7.1 hook の数字と stats(画面下4項目)は意味的に一致させる

- hookが「チームのバント激減=大正解」 → stats=犠打数/得点/勝率/勝ち越し
- hookが「個人のIsoP急上昇」 → stats=AVG/OPS/HR/RBI

★ hookとstatsが**矛盾するとミーハー脱落** (例: hook「大正解」なのに stats AVG.228 → 何の話?)

## 8. id:1 の playerType / silhouetteType 整合性

| テーマ | playerType | silhouetteType | hook例 |
|---|---|---|---|
| 投手分析 | pitcher | pitcher_left/right/set/catcher | 「【井上温大】\n奪三振【11.2】」 |
| 打者分析 | batter | batter_left/right/stance/runner | 「【岡本和真】\n打率.228でも『主軸』」 |
| チーム/監督/采配 | team | team_stadium / team_huddle | 「巨人の犠打\n【80→18】激減」 |

★ 監督ネタで打者シルエットだと**「誰?」で離脱**。テーマと型を一致させる。

## 9. ハッシュタグと固定コメント (動画投稿時の補助)

### 9.1 ハッシュタグ
基本: `#巨人 #ジャイアンツ #プロ野球データ #数字で見るG党`
+ 動画固有のもの: `#選手名` `#試合名` (例: `#井上温大` `#岡本和真`)

### 9.2 固定コメント
動画末尾の二択疑問に視聴者を呼び込む形:
```
今日の井上、皆さんは「三振の山を信じる派」or「天敵・佐野が心配派」？
コメント欄でぜひ予想を教えてください👇
```

★ コメント率を上げるための補助。動画本体には含めない (アプリ生成 JSON とは別)。

## 10. ★v5.20.10★ id:1 の speech 短縮テクニック

### 原則
text は完全形 (3-4行 × 6-8字) だが、**speech はテンポ優先で短く切ってよい**。

### 理由
- 動画開始 2 秒以内が「スクロール継続/離脱」の分岐点
- text 全文を読み上げると 4-5 秒かかり、その間に離脱されるリスク
- **speech は印象的なフレーズだけ** (1-2 秒で言える 8-15 文字程度) で十分
- 残りは text のビジュアルインパクトで伝える

### 例

text 全文:
```
岡本和真
【.305】
覚醒の理由
```

❌ NG (長すぎ):
```json
"speech": "おかもとかずまの今季の打率はさんわりごぶ、覚醒の理由を解説します。"
```

✅ OK (短く印象的):
```json
"speech": "岡本和真。【.305】の覚醒。"
```

### 判断基準
- speech は **2 秒以内に終わる長さ** が目安 (約 12-15 文字)
- 主語 (固有名詞) + 核心ワードだけ
- 説明調の動詞は省く
- 「。」で間を作って印象づけ

## 11. ★v5.20.10★ id:1 のギャップ作り (即スワイプ防止)

### 鉄則: id:1 で全結論を言わない

冒頭で**結論まで提示すると「もう知った、離脱」**となる。

### ギャップ・謎を残す

❌ NG: 「岡本が3割打者になった理由は配球変化でした」
   → 結論が見えて離脱

✅ OK: 「岡本和真【.305】覚醒の理由」
   → 「なぜ?」と思わせて続きを見たくなる

✅ OK: 「戸郷翔征 奪三振封印!?」
   → 通常イメージ (奪三振王) と逆 → 「なぜ?」

### 序盤の最速回収 (期待値コントロール)

id:1 で提示した謎の**核心データ**は、**id:4-8 までに最速で提示**:

```
id:1 「奪三振封印!?」
id:2-3 状況設定
id:4 「奪三振率は前年比 -3.2」 ← ここで核心提示 (期待回収)
id:5-30 多角的に深掘り
```

❌ NG: id:1 で「奪三振封印!?」と言いながら、id:8 まで打率の話 → 期待ズレで離脱
