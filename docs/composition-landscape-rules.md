# 横長動画(16:9)専用ルール (composition-landscape-rules.md)

★ Gem が動画タイプ「横長 16:9」を選定した時に参照する。
★ ショート(9:16)用 `composition-rules.md` とは設計思想が **根本的に違う**。

---

## 1. 横長動画の本質的特性

| 項目 | 9:16 ショート | 16:9 横長 |
|---|---|---|
| 視聴環境 | スマホ縦持ち、フリック | PC / TV / 横持ち、集中視聴 |
| スワイプ概念 | あり(0.5 秒判定) | なし(自動再生) |
| 動画長 | 60 秒未満 | 60 秒〜数分(主流 2-5 分) |
| 求められる動画タイプ | 「謎・矛盾」短時間刺激 | 「解説・深掘り」体系的理解 |
| キャラ配置 | 中央 + テロップ縦長 | 横並び + テロップ横長 |
| 派手な演出 | 必要(スワイプ防止) | 落ち着いた方が好まれる |

---

## 2. 章立て構造(★最重要)

横長動画はショートと違い**章立て**が機能する。

### 標準: 5 章構造(2-3 分動画)

```
【オープニング】 (3-5 ID)
  - 議題の前提共有 + キャラ呼び合い

【章 1: 状況設定】
  - 選手の現在の状況、ファン心理

【章 2: 核心データ】
  - 数字の山場 1 (最強の数字を最速で見せる)

【章 3: 深掘り・対比】
  - 数字の山場 2 (他指標との関係)

【章 4: 根本原因】
  - アハ体験、権威の引用

【エンディング】 (3-5 ID)
  - まとめ + 登録誘導
```

### 動画長別の目安

| 動画長 | 推奨章数 |
|---|---|
| 1-2 分 | 4 章 |
| 2-3 分 | 5 章 |
| 3-5 分 | 6-7 章 |
| 5 分超 | 8 章以上 |

★ id 数の固定指定はしない。内容に応じて柔軟に。

### 章間の演出
- 章の切替で `transition_swoosh` 効果音
- 章ヘッダー ID(text=「【章 N: タイトル】」)を 1 つ使う

---

## 3. id:1(オープニング)の設計

★ ショート用 4 戦略(A/B/C/D)は**横長では使わない**。スワイプ防止が不要なため。

### 横長 id:1 パターン

| パターン | 例 |
|---|---|
| **宣言型** | 「○○選手の今季成績、データで読み解きます」 |
| **問いかけ型** | 「○○、ここまでの活躍は本物か?データで検証」 |
| **前提共有型** | 「数字で見る○○の真価 — これを知らずに語れない」 |
| **時系列型** | 「2024-2026 ○○選手の変遷を 1 つずつ追います」 |

- タイトル長: **20-40 字**(ショート 25-32 字より広い)
- textSize: `l`(ショート xl ほど大きくない)
- scenePreset: `default` / `cinematic_zoom`
- se: `transition_swoosh` / `hook_impact`(派手すぎない)

---

## 4. レイアウト推奨(章ごと)

| 章 | 推奨レイアウト |
|---|---|
| オープニング | `player_spotlight` |
| 状況設定 | `timeline` / `team_context` |
| 核心データ | `radar_compare` |
| 深掘り・対比 | `versus_card` / `ranking` |
| 根本原因 | `pitch_arsenal` / `batter_heatmap` |
| エンディング | `player_spotlight` |

★ ただし projectData.layoutType は単一値。章ごとのレイアウト切替は Gem が判断する範囲外なので、Gem では「主役レイアウト」1 つを推奨するに留める。

---

## 5. テロップ(text プロパティ)

- 1 ID あたり **8-20 字 × 1-2 行**(ショートの 3-12 字×3-4 行とは違う)
- 横長配置、文字大きく明瞭
- 句点「。」NG(ショート/横長共通)
- 数値は【】、指標名は「」、衝撃ワードは『』

---

## 6. ハイライト指標(comparisons)

横長では **5-10 個**(ショートの 5-7 個より多め)。
- 章ごとに 2-3 個ずつ紹介
- variants[] で「対左/対右」等のスコープ切替を活用

---

## 7. NG ワード(ショート共通)

詳細は composition-rules.md §6 参照。

横長特有の注意:
- 長尺だと「飽和パワーワード(致命的/絶望/衝撃)」連発で視聴者が疲れる
- 章ごとに 1-2 回まで

---

## 8. Gem の横長設計時の中間判断例

```
【横長 16:9・5 章構造・角度: 矛盾深掘り+起用事情ハイブリッド】

採用タイトル(id:1):
データで読み解く戸郷翔征の現在地。質の高い直球とフォークの課題

採用データ:
- 防御率 7.20 / 2024年 1.95 比較
- xFIP 3.39(実力ベース)
- Stuff+ 107.3(直球の威力)
- Chase% 35.3%(誘い込み健在)
- HardHit% 75.0%(★衝撃データ)
- Location+ 95.4(制球課題)
- 5/12 広島戦 5回 110球

Gem の指示:
1. 5 章構造(オープニング + 状況設定/核心データ/深掘り/根本原因 + エンディング)
2. 章切替に transition_swoosh、章ヘッダー ID は textSize: l / scenePreset: mono_drama
3. 主役レイアウト: player_spotlight (projectData.layoutType)
4. もえか「ヤバい」運用は HardHit% 75% の章で 1 回のみ(疑問形+敬語)
5. 監督批判は層批判に変換(「ローテ全体」)
6. アウトロ: 次回登板の見どころ提示 + 登録誘導
```
